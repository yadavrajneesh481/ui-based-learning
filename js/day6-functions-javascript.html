Functions Kya Hain? - Complete Deep Dive

Functions JavaScript mein first-class citizens hain - matlab yeh values ki tarah treat hote hain. Functions reusable code blocks hain jo specific tasks perform karte hain, parameters accept kar sakte hain, aur values return kar sakte hain. Yeh code organization, reusability, aur modularity ke liye essential hain.

Function Fundamentals aur Concepts:

üß† Core Function Concepts:

  • First-Class Functions: Variables mein store kar sakte hain, parameters mein pass kar sakte hain
  • Higher-Order Functions: Functions jo dusre functions ko accept ya return karte hain
  • Pure Functions: Same input ke liye same output, no side effects
  • Function Scope: Functions apna local scope create karte hain
  • Closures: Inner functions outer function ke variables access kar sakte hain

Function Declaration Methods - Detailed Comparison:

üîµ Function Declaration

function functionName(parameters) {
  // function body
  return value;
}

Hoisting: ‚úÖ Fully hoisted

this Binding: Dynamic

Constructor: ‚úÖ Can be used with 'new'

Use Case: Main functions, methods

üü¢ Function Expression

const functionName = function(parameters) {
  // function body
  return value;
};

Hoisting: ‚ùå Variable hoisted, function not

this Binding: Dynamic

Constructor: ‚úÖ Can be used with 'new'

Use Case: Conditional functions, callbacks

üü° Arrow Function (ES6)

const functionName = (parameters) => {
  // function body
  return value;
};
// Short: const fn = x => x * 2;

Hoisting: ‚ùå Not hoisted

this Binding: Lexical (inherited)

Constructor: ‚ùå Cannot use with 'new'

Use Case: Callbacks, short functions

üî¥ IIFE (Immediately Invoked)

(function(parameters) {
  // function body
  // executes immediately
})(arguments);
// Arrow IIFE: (() => {})();

Hoisting: N/A (executes immediately)

this Binding: Depends on type

Constructor: ‚ùå Cannot use

Use Case: Module pattern, avoid pollution

Function Composition aur Currying:

üîó Function Composition:

Multiple functions ko combine karke complex operations banana:

const add = x => y => x + y;
const multiply = x => y => x * y;
const compose = (f, g) => x => f(g(x));

const addThenMultiply = compose(multiply(2), add(3));
console.log(addThenMultiply(5)); // (5 + 3) * 2 = 16

Interview Questions

1. Basic: Function declaration aur function expression mein kya difference hai?

Answer: Declaration hoisted hoti hai (pehle use kar sakte hain), expression nahi. Declaration: function name() {}, Expression: const name = function() {}

2. Basic: Arrow function kya hai aur regular function se kya different hai?

Answer: ES6 syntax, shorter hai. Main difference: 'this' binding different hai, hoisting nahi hoti, constructor nahi ban sakta.

3. Intermediate: Default parameters kya hain? Example do.

Answer: Function parameters ki default values. Example: function greet(name = "Guest") { return "Hello " + name; }

4. Intermediate: Rest parameters (...args) kya karte hain?

Answer: Variable number of arguments ko array mein collect karte hain. function sum(...numbers) { return numbers.reduce((a,b) => a+b); }

5. Advanced: Function hoisting kya hai? Arrow functions hoist hote hain?

Answer: Function declarations top pe move ho jaati hain. Arrow functions hoist nahi hote kyunki yeh variables mein store hote hain.

6. Advanced: Higher-order functions kya hain? Example do.

Answer: Functions jo dusre functions ko parameter mein lete hain ya return karte hain. Example: map, filter, reduce.
