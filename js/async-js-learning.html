1Ô∏è‚É£ Synchronous vs Asynchronous Execution

Synchronous Execution kya hai?

Synchronous matlab ek line-by-line execution. Jab tak ek task complete nahi hota, next task wait karta rehta hai. Ye bilkul queue jaisa hai - ek ke baad ek.

Example: Aap line mein khade ho ticket lene ke liye. Aapke aage wala person jab tak ticket nahi leta, aapki baari nahi aati.

Asynchronous Execution kya hai?

Asynchronous matlab tasks parallel mein chal sakte hain. Ek task complete hone ka wait kiye bina, dusra task shuru ho sakta hai.

Example: Restaurant mein aapne order kiya, chef cooking kar raha hai, aur aap waiter se baat kar sakte ho simultaneously.

2Ô∏è‚É£ Callback Functions

Callback kya hota hai?

Callback ek function hai jo dusre function ko argument ke tarah pass kiya jata hai. Ye function baad mein execute hota hai jab koi specific event ho jaye ya task complete ho jaye.

Simple words mein: "Ye kaam kar, aur jab ho jaye toh mujhe batana" - ye batana wala part callback hai.

// Basic Callback Example function doSomething(callback) { console.log("Kaam shuru..."); setTimeout(() => { console.log("Kaam khatam!"); callback(); // Callback function call }, 2000); } doSomething(() => { console.log("Thank you!"); });

setTimeout

setTimeout ek function ko specified delay ke baad sirf ek baar execute karta hai.

Syntax: setTimeout(callback, delayInMilliseconds)

// setTimeout Example console.log("Start"); setTimeout(() => { console.log("2 seconds baad ye print hoga"); }, 2000); console.log("End"); // Output: Start ‚Üí End ‚Üí 2 seconds baad ye print hoga

setInterval

setInterval ek function ko specified interval par repeatedly execute karta hai jab tak aap use stop na kar do.

Syntax: setInterval(callback, intervalInMilliseconds)

üìù Interview Questions

Q1. Synchronous aur Asynchronous execution mein kya difference hai? Easy
Answer: Synchronous execution mein code line-by-line execute hota hai aur ek statement complete hone ka wait karta hai. Asynchronous execution mein code wait nahi karta aur non-blocking hota hai. Example: setTimeout asynchronous hai kyunki ye timer set karke immediately next line par chala jata hai.
Q2. Callback function kya hota hai? Example ke saath explain karo. Easy
Answer: Callback ek function hai jo dusre function ko argument ke taur par pass kiya jata hai aur baad mein execute hota hai. Example:

function greet(name, callback) {
  console.log('Hello ' + name);
  callback();
}
greet('John', function() { console.log('Callback executed'); });
Q3. setTimeout(fn, 0) kya karta hai aur kyun use hota hai? Medium
Answer: setTimeout(fn, 0) function ko immediately execute nahi karta, balki use callback queue mein daal deta hai. Ye tab useful hai jab aap chahte ho ki current execution stack complete ho jaye, phir callback execute ho. Isse event loop ka proper use hota hai aur UI blocking se bacha ja sakta hai.
Q4. Callback Hell kya hai aur isse kaise avoid karein? Medium
Answer: Callback Hell tab hota hai jab multiple nested callbacks use karte hain, jo code ko unreadable aur unmaintainable bana deta hai. Isse avoid karne ke liye:
1. Named functions use karein instead of anonymous functions
2. Promises use karein
3. Async/Await use karein
4. Code ko modular banayein
Q5. setTimeout aur setInterval mein kya difference hai? Easy
Answer: setTimeout ek function ko specified delay ke baad sirf ek baar execute karta hai. setInterval ek function ko repeatedly specified interval par execute karta rehta hai jab tak clearInterval() call na ho. Example: setTimeout(fn, 1000) - 1 second baad ek baar run hoga. setInterval(fn, 1000) - har 1 second mein run hota rahega.
Q6. clearTimeout aur clearInterval kab aur kyun use karte hain? Medium
Answer: clearTimeout() aur clearInterval() ko use karte hain jab hume scheduled callbacks ko cancel karna ho. Ye memory leaks prevent karne ke liye important hai. Example:

let timerId = setTimeout(fn, 5000);
clearTimeout(timerId); // Cancel kar diya

let intervalId = setInterval(fn, 1000);
clearInterval(intervalId); // Stop kar diya

Especially important when component unmount ho raha ho (React/Vue mein).
Q7. Event Loop kya hai aur ye callbacks ko kaise handle karta hai? Hard
Answer: Event Loop JavaScript ka core mechanism hai jo asynchronous operations handle karta hai. Process:
1. Call Stack mein synchronous code execute hota hai
2. Async operations (setTimeout, callbacks) Web APIs ko diye jate hain
3. Complete hone par callbacks Task Queue mein aate hain
4. Jab Call Stack empty hota hai, Event Loop Task Queue se callbacks uthake Call Stack mein dalta hai
5. Phir callbacks execute hote hain

Isse JavaScript single-threaded hote hue bhi non-blocking behavior achieve karta hai.
