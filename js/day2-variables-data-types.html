Variables Kya Hain? - Complete Deep Dive

Variables programming mein containers hain jo data store karte hain. Yeh memory locations hain jahan hum values rakh sakte hain aur baad mein use kar sakte hain. JavaScript mein variables declare karne ke teen main tarike hain, har ek ka apna behavior hai.

Variable Declaration Evolution:

üìö Historical Context:

  • ES5 (2009): Sirf var available tha
  • ES6 (2015): let aur const introduce hue
  • Modern JavaScript: var avoid karte hain, let/const prefer karte hain

Variable Declaration Types - Detailed Comparison:

üî¥ var (Legacy)

Scope: Function-scoped

Hoisting: Yes, initialized with undefined

Redeclaration: Allowed

Reassignment: Allowed

Use: Avoid in modern code

üü¢ let (Modern)

Scope: Block-scoped

Hoisting: Yes, but in TDZ

Redeclaration: Not allowed

Reassignment: Allowed

Use: For variables that change

üîµ const (Constant)

Scope: Block-scoped

Hoisting: Yes, but in TDZ

Redeclaration: Not allowed

Reassignment: Not allowed

Use: For constants & objects

Variable Naming Rules aur Best Practices

JavaScript mein variables ka naam rakhne ke specific rules aur conventions hain:

Naming Rules (Mandatory):

  • Letters (a-z, A-Z), digits (0-9), underscore (_), dollar sign ($) allowed
  • Cannot start with a digit
  • Case-sensitive (myVar ‚â† MyVar ‚â† MYVAR)
  • Cannot use reserved keywords (let, const, function, etc.)
  • No spaces allowed

Naming Conventions (Best Practices):

‚úÖ Good Examples

  • userName (camelCase)
  • totalPrice
  • isLoggedIn (boolean)
  • MAX_USERS (constants)
  • _privateVar (private)

‚ùå Bad Examples

  • 2names (starts with digit)
  • user name (space)
  • let (reserved keyword)
  • a (not descriptive)
  • user_name (snake_case)

Interview Questions

1. Basic: var, let, aur const mein kya difference hai?

Answer: var function-scoped hai aur redeclare ho sakta hai, let block-scoped hai, const bhi block-scoped hai lekin reassign nahi ho sakta.

2. Basic: JavaScript mein kitne primitive data types hain?

Answer: 7 primitive types: string, number, boolean, null, undefined, symbol, bigint.

3. Intermediate: null aur undefined mein kya difference hai?

Answer: null explicitly assigned empty value hai, undefined variable declare hua hai lekin value assign nahi hui.

4. Intermediate: Type coercion kya hai? Example do.

Answer: Automatic type conversion. Example: "5" + 3 = "53" (string), "5" - 3 = 2 (number).

5. Advanced: typeof null kya return karta hai aur kyun?

Answer: "object" return karta hai. Yeh JavaScript ka historical bug hai jo backward compatibility ke liye fix nahi kiya gaya.
