üìö Objects Kya Hote Hain?

Objects JavaScript mein ek fundamental data structure hai jo related data aur functionality ko ek saath store karta hai. Yeh real-world entities ko represent karne ka best tareeka hai. JavaScript mein almost everything ek object hai!

Example: Ek student ko represent karne ke liye - uska name, age, rollNumber sab ek jagah store kar sakte hain.

üß† Object Deep Concepts:

  • Key-Value Pairs: Objects properties (keys) aur unki values store karte hain
  • Reference Type: Objects memory mein reference store karte hain, values nahi
  • Mutable: Object properties ko runtime pe change kar sakte hain
  • Prototype Chain: Objects inherit properties from prototype
  • Dynamic Properties: Runtime pe properties add/remove kar sakte hain
  • Methods: Objects mein functions bhi store kar sakte hain

üéØ The "this" Keyword

"this" keyword current object ko refer karta hai. Yeh context-dependent hai - matlab different situations mein different values refer kar sakta hai.

Object method mein: "this" us object ko point karta hai jis object ka method call ho raha hai.

const person = { name: "Sneha", age: 25, greet: function() { // 'this' person object ko refer karega return "Hello, I am " + this.name + " and I am " + this.age + " years old"; } }; console.log(person.greet());

üîß Object.keys(), Object.values(), Object.entries()

Yeh teen built-in methods hain jo objects ke saath kaam karne mein madad karte hain:

Object.keys()

Object ke saare property names (keys) ka array return karta hai.

const car = { brand: "Honda", model: "City", year: 2023 }; console.log(Object.keys(car)); // Output: ["brand", "model", "year"]

Object.values()

Object ke saare property values ka array return karta hai.

console.log(Object.values(car)); // Output: ["Honda", "City", 2023]

Object.entries()

Har property ke liye [key, value] pairs ka array return karta hai.

console.log(Object.entries(car)); // Output: [["brand", "Honda"], ["model", "City"], ["year", 2023]]

üé§ Interview Questions

Q1 What is the difference between dot notation and bracket notation for accessing object properties?

Dot Notation: Used when property name is a valid identifier (no spaces, doesn't start with number). Example: obj.name

Bracket Notation: Used when property name has spaces, starts with number, or is stored in a variable. Example: obj["first name"] or obj[variable]

Hinglish: Dot notation simple cases ke liye use hota hai jab property name straightforward ho. Bracket notation flexible hai aur dynamic property access ke liye useful hai.

Q2 Explain the "this" keyword in JavaScript objects.

"this" refers to the object that is executing the current function. In object methods, "this" refers to the object that owns the method.

Example: If person.greet() is called, "this" inside greet() refers to the person object.

Important: Arrow functions don't have their own "this" - they inherit from parent scope.

Q3 What is the difference between Object.keys(), Object.values(), and Object.entries()?

Object.keys(): Returns array of property names (keys)

Object.values(): Returns array of property values

Object.entries(): Returns array of [key, value] pairs

Use case: Keys for checking property existence, values for processing data, entries for complete iteration with both key and value.

Q4 How do you create multiple similar objects efficiently?

Method 1: Constructor Functions

function Person(name, age) {
    this.name = name;
    this.age = age;
}
const person1 = new Person("Raj", 25);

Method 2: ES6 Classes (preferred modern approach)

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}

Method 3: Factory Functions

function createPerson(name, age) {
    return { name, age };
}
Q5 How do you check if a property exists in an object?

Method 1: "in" operator

'name' in person // returns true if exists

Method 2: hasOwnProperty()

person.hasOwnProperty('name') // checks own properties only

Method 3: Direct check

person.name !== undefined

Difference: "in" checks inherited properties too, hasOwnProperty checks only own properties.

Q6 What happens when you use "delete" on an object property?

delete operator removes a property from an object completely (both key and value).

Returns: true if deletion is successful, false if property can't be deleted

Important: delete only works on object properties, not variables. Can't delete properties from prototype chain.

const obj = { name: "Raj", age: 25 };
delete obj.age; // Returns true
console.log(obj); // { name: "Raj" }
Q7 Explain shallow copy vs deep copy of objects.

Shallow Copy: Copies only first level properties. Nested objects are still referenced.

const shallow = Object.assign({}, original);
// or
const shallow = {...original};

Deep Copy: Copies all levels recursively. Nested objects are also cloned.

const deep = JSON.parse(JSON.stringify(original));
// Note: This doesn't work with functions, dates, undefined

Hinglish: Shallow copy surface level copy hai, deep copy complete copy hai jismein nested objects bhi alag se copy ho jaate hain.
