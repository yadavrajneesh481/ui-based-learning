üìö Introduction - Fetch API Kya Hai?

Fetch API ek modern JavaScript feature hai jo humein server se data fetch karne ki capability deta hai. Ye promise-based hai, matlab ye asynchronous operations ko handle karna bahut easy bana deta hai.

AJAX Kya Hai?

AJAX (Asynchronous JavaScript And XML) ek technique hai jisse hum bina page reload kiye server se data exchange kar sakte hain. Modern web applications mein ye bahut important hai.

// Basic Fetch Syntax fetch('https://api.example.com/data') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));

üîë Important Concepts

1. Response Object

Fetch request ka result ek Response object hota hai jismein:

  • response.ok - Success status check karna (200-299)
  • response.status - HTTP status code (404, 500, etc.)
  • response.json() - JSON data extract karna
  • response.text() - Text data extract karna

2. HTTP Methods

  • GET - Data fetch karna (default)
  • POST - Naya data create karna
  • PUT - Existing data update karna
  • DELETE - Data delete karna

3. Headers

Headers request ke sath additional information bhejte hain:

const headers = { 'Content-Type': 'application/json', 'Authorization': 'Bearer token123' };

üíº Interview Questions

Easy

Q1: Fetch API aur XMLHttpRequest mein kya difference hai?

Answer:

  • Fetch API promise-based hai, modern aur cleaner syntax hai
  • XMLHttpRequest callback-based hai, purana approach hai
  • Fetch mein error handling better hai aur code readability zyada hai
  • Fetch by default cookies nahi bhejta unless credentials option set ho
Easy

Q2: response.json() method kya karta hai?

Answer:

response.json() method response body ko read karta hai aur usse JavaScript object mein parse karta hai. Ye ek promise return karta hai jo resolve hota hai parsed JSON data ke sath.

Medium

Q3: Fetch API mein error handling kaise karte hain? Response.ok kab false hota hai?

Answer:

Fetch API automatically errors ko reject nahi karta jab tak network failure na ho. response.ok property false hoti hai jab HTTP status code 200-299 range se bahar ho (404, 500, etc.). Isliye manually check karna padta hai:

if (!response.ok) {
    throw new Error('HTTP Error: ' + response.status);
}
Medium

Q4: CORS kya hai aur fetch requests mein ye kaise affect karta hai?

Answer:

CORS (Cross-Origin Resource Sharing) ek security mechanism hai jo browsers mein implement hota hai. Ye control karta hai ki ek domain se dusre domain ko requests kab allowed hain. Agar server CORS headers set nahi karta, to browser request ko block kar deta hai. Developer ko server-side CORS configuration karna padta hai ya proxy use karni padti hai.

Hard

Q5: Async/await aur .then()/.catch() mein kya difference hai? Kab kya use karna chahiye?

Answer:

  • Async/await: Synchronous jaise code likhne ka tarika, error handling try-catch se hoti hai, code zyada readable hai
  • .then()/.catch(): Promise chains banate hain, multiple parallel operations ke liye better hai
  • Use Case: Sequential operations ke liye async/await better hai, parallel operations ke liye Promise.all() with .then() better hai
Hard

Q6: Multiple API calls ko efficiently kaise handle karenge? Promise.all() vs Promise.race() explain karein.

Answer:

  • Promise.all(): Sabhi promises complete hone ka wait karta hai. Agar ek bhi fail ho to pura reject ho jata hai. Use karein jab sabhi responses chahiye hon.
  • Promise.race(): Pehla promise jo complete ho jaye uska result return karta hai. Use karein jab fastest response chahiye ho (e.g., multiple servers se ek hi data fetch karna).
  • Promise.allSettled(): Sabhi promises complete hone ka wait karta hai chahe fail ho ya succeed, har ek ka status return karta hai.
Hard

Q7: Fetch API mein request cancellation kaise implement karenge?

Answer:

AbortController use karke request cancel kar sakte hain:

const controller = new AbortController();
const signal = controller.signal;

fetch(url, { signal })
    .then(response => response.json())
    .catch(error => {
        if (error.name === 'AbortError') {
            console.log('Request cancelled');
        }
    });

// Cancel request
controller.abort();

Ye useful hai jab user navigate away kare ya search input rapidly change ho.
