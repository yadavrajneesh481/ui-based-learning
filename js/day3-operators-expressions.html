Arithmetic Operators - Complete Deep Dive

Arithmetic operators mathematical calculations perform karte hain. JavaScript mein yeh operators numbers ke saath kaam karte hain aur automatic type conversion bhi karte hain jab zaroorat ho.

Operator Types aur Unka Behavior:

üìä Mathematical Operations:

  • Binary Operators: Do operands chahiye (a + b)
  • Unary Operators: Ek operand chahiye (+a, -a)
  • Type Coercion: Strings ko numbers mein convert kar dete hain
  • Special Values: Infinity, -Infinity, NaN handle karte hain

Detailed Operator Analysis:

‚ûï Addition (+)

Numbers: 5 + 3 = 8

Strings: "Hello" + " World" = "Hello World"

Mixed: "5" + 3 = "53" (string concatenation)

Special: true + 1 = 2

‚ûñ Subtraction (-)

Numbers: 10 - 3 = 7

Strings: "10" - "3" = 7 (converts to numbers)

Special: "hello" - 5 = NaN

Unary: -5 (negation)

‚úñÔ∏è Multiplication (*)

Numbers: 4 * 5 = 20

Strings: "4" * "5" = 20

Special: Infinity * 0 = NaN

Boolean: true * 5 = 5

‚ûó Division (/)

Numbers: 15 / 3 = 5

Zero: 5 / 0 = Infinity

Zero by Zero: 0 / 0 = NaN

Negative: -10 / 2 = -5

üìê Modulus (%)

Remainder: 10 % 3 = 1

Even/Odd: n % 2 === 0 (even check)

Negative: -10 % 3 = -1

Float: 5.5 % 2 = 1.5

üî¢ Exponentiation (**)

Power: 2 ** 3 = 8

Square Root: 9 ** 0.5 = 3

Negative: 2 ** -2 = 0.25

Interview Questions

1. Basic: == aur === mein kya difference hai?

Answer: == type coercion karta hai, === strict comparison karta hai bina type conversion ke.

2. Basic: Modulus operator (%) kya karta hai?

Answer: Division ka remainder return karta hai. Example: 7 % 3 = 1.

3. Intermediate: 2 + 3 * 4 ka result kya hoga aur kyun?

Answer: 14, kyunki multiplication ka precedence addition se higher hai. Pehle 3*4=12, phir 2+12=14.

4. Intermediate: Logical operators mein short-circuit evaluation kya hai?

Answer: && mein agar pehla false hai to dusra check nahi karta, || mein agar pehla true hai to dusra check nahi karta.

5. Advanced: typeof null kya return karta hai aur yeh bug kyun hai?

Answer: "object" return karta hai. Yeh JavaScript ka historical bug hai jo backward compatibility ke liye fix nahi kiya gaya.
