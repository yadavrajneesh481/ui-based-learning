Getters and Setters

Getters: Property ko read karte waqt function execute hota hai

Setters: Property ko set karte waqt function execute hota hai

const person = {
    firstName: "Amit",
    lastName: "Sharma",
    
    get fullName() {
        return this.firstName + " " + this.lastName;

        this.firstName = parts[0];
        this.lastName = parts[1];

};

person.fullName = "Raj Kumar";  // (setter)
console.log(person.firstName);  // "Raj"

function Car(brand, model) {
    this.brand = brand;
    this.model = model;
}

// Adding method to prototype
Car.prototype.getInfo = function() {
    return this.brand + " " + this.model;

Proxy Objects

Proxy ek wrapper hai jo object ke operations ko intercept kar sakta hai aur custom behavior add kar sakta hai.

const target = {
    name: "Rahul",
    age: 25
};

const handler = {
    get(target, property) {
        console.log(`Getting ${property}`);
        return target[property];
    },
    set(target, property, value) {
        console.log(`Setting ${property} to ${value}`);
        target[property] = value;
        return true;
    }
};

const proxy = new Proxy(target, handler);
console.log(proxy.name); // Logs: "Getting name", returns "Rahul"
proxy.age = 26; // Logs: "Setting age to 26"

üíº Interview Questions & Answers

Question 1 Basic
What is the difference between dot notation and bracket notation for accessing object properties?
Answer:
Dot Notation: object.property
- Simple aur readable
- Property name valid identifier hona chahiye
- Dynamic property names ke saath kaam nahi karta

Bracket Notation: object["property"]
- Property name string ya variable ho sakta hai
- Special characters aur spaces allow hain
- Dynamic property access ke liye use hota hai

Example:
const obj = { "first name": "Rahul" };
// obj.first name; // Error!
obj["first name"]; // Works!

const key = "age";
obj[key]; // Dynamic access
Question 2 Basic
Explain the difference between Object.freeze() and Object.seal().
Answer:

Object.freeze():
- Properties add/delete/modify - sab kuch block
- Object completely immutable ban jata hai
- Existing properties ki values change nahi ho sakti

Object.seal():
- Properties add/delete nahi kar sakte
- Lekin existing properties modify kar sakte hain
- Property descriptors change nahi ho sakte

Example:
const frozen = Object.freeze({ a: 1 });
frozen.a = 2; // Won't work
frozen.b = 3; // Won't work

const sealed = Object.seal({ a: 1 });
sealed.a = 2; // Works! ‚úÖ
sealed.b = 3; // Won't work ‚ùå
Question 3 Intermediate
What is the difference between shallow copy and deep copy? How do you create each?
Answer:

Shallow Copy:
- Sirf top-level properties copy hoti hain
- Nested objects ka reference copy hota hai
- Original nested object change hoga toh copy bhi change hoga

Methods:
const obj = { a: 1, b: { c: 2 } };

// Spread operator
const copy1 = { ...obj };

// Object.assign()
const copy2 = Object.assign({}, obj);

Deep Copy:
- Puri object recursively copy hoti hai
- Nested objects bhi completely copy hote hain
- Original aur copy completely independent

Methods:
// JSON method (simple objects ke liye)
const deepCopy1 = JSON.parse(JSON.stringify(obj));

// structuredClone (modern browsers)
const deepCopy2 = structuredClone(obj);
Question 4 Intermediate
Explain the 'this' keyword in JavaScript objects. How does its value change in different contexts?
Answer:

'this' keyword current execution context ko refer karta hai. Iska value depend karta hai ki function kaise call hua.

1. Object Method:
const obj = {
  name: "Rahul",
  greet() { return this.name; }
};
obj.greet(); // 'this' = obj

2. Regular Function:
function show() { console.log(this); }
show(); // 'this' = window (browser) / global (Node)

3. Arrow Function:
const obj = {
  name: "Rahul",
  greet: () => this.name
};
// Arrow function mein 'this' lexical scope se aata hai

4. Constructor:
function Person(name) {
  this.name = name; // 'this' = new object
}
Question 5 Advanced
What is the prototype chain in JavaScript? How does property lookup work?
Answer:

Prototype Chain ek mechanism hai jisse objects properties aur methods inherit karte hain.

Property Lookup Process:
1. Pehle object mein property dhundho
2. Agar nahi mili, toh object ke prototype mein dhundho
3. Phir prototype ke prototype mein dhundho
4. Yeh chain tab tak chalti hai jab tak null nahi mil jata

Example:
function Animal(name) {
  this.name = name;
}

Animal.prototype.speak = function() {
  return this.name + " makes sound";
};

const dog = new Animal("Tommy");

// Lookup chain:
// dog.speak() ‚Üí
// 1. dog object mein 'speak' nahi hai
// 2. dog.__proto__ (Animal.prototype) mein mil gaya!
// 3. Execute ho gaya

console.log(dog.__proto__ === Animal.prototype); // true
console.log(Animal.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null
Question 6 Advanced
Explain Object.defineProperty() and property descriptors. What are writable, enumerable, and configurable?
Answer:

Object.defineProperty() se hum property ke behavior ko fine-tune kar sakte hain using descriptors.

Property Descriptors:

1. writable: Property ki value change ho sakti hai ya nahi
Object.defineProperty(obj, 'name', {
  value: 'Rahul',
  writable: false // Value change nahi hogi
});

2. enumerable: Property loops mein dikhegi ya nahi
Object.defineProperty(obj, 'secret', {
  value: 'hidden',
  enumerable: false // for...in mein nahi dikhega
});

3. configurable: Property delete ho sakti hai ya descriptor change ho sakta hai
Object.defineProperty(obj, 'id', {
  value: 123,
  configurable: false // Delete nahi kar sakte
});

Complete Example:
const user = {};

Object.defineProperty(user, 'name', {
  value: 'Rahul',
  writable: true,
  enumerable: true,
  configurable: true
});

// Get descriptor
const descriptor = Object.getOwnPropertyDescriptor(user, 'name');
console.log(descriptor);
Question 7 Advanced
What are Proxy and Reflect in JavaScript? Provide practical use cases.
Answer:

Proxy: Ek wrapper object jo fundamental operations ko intercept kar sakta hai.

Reflect: Built-in object jo object operations ke liye methods provide karta hai.

Practical Use Cases:

1. Validation:
const validator = {
  set(target, property, value) {
    if (property === 'age') {
      if (typeof value !== 'number') {
        throw new TypeError('Age must be number');
      }
      if (value < 0) {
        throw new RangeError('Age cannot be negative');
      }
    }
    target[property] = value;
    return true;
  }
};

const person = new Proxy({}, validator);
person.age = 25; // Works
// person.age = -5; // Error!

2. Logging/Debugging:
const handler = {
  get(target, property) {
    console.log(`Accessing ${property}`);
    return Reflect.get(target, property);
  }
};

3. Default Values:
const withDefaults = new Proxy({}, {
  get(target, property) {
    return property in target ? target[property] : 'N/A';
  }
});
