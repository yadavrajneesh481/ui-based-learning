Webpack - Module Bundler

Webpack multiple JavaScript files ko ek ya multiple bundles mein combine karta hai. Yeh dependencies resolve karta hai aur assets optimize karta hai.

Webpack Configuration (webpack.config.js):

const path = require('path'); module.exports = { entry: './src/index.js', output: { path: path.resolve(__dirname, 'dist'), filename: 'bundle.js' }, module: { rules: [ { test: /\.js$/, exclude: /node_modules/, use: { loader: 'babel-loader' } }, { test: /\.css$/, use: ['style-loader', 'css-loader'] }, { test: /\.(png|jpg|gif)$/, use: ['file-loader'] } ] }, plugins: [ new HtmlWebpackPlugin({ template: './src/index.html' }) ], devServer: { contentBase: './dist', hot: true } };

Vite - Fast Build Tool

Vite modern build tool hai jo development mein bahut fast hai. Yeh ES modules use karta hai aur HMR (Hot Module Replacement) provide karta hai.

Practice: Simple Build Pipeline

Ab ek complete build pipeline setup karenge jo development se production tak ka workflow handle kare.

Build Pipeline Steps:

  1. Development: Source code with ES6+ features
  2. Linting: ESLint checks for code quality
  3. Formatting: Prettier formats the code
  4. Transpilation: Babel converts ES6+ to ES5
  5. Bundling: Webpack/Vite creates optimized bundles
  6. Minification: Code size optimization
  7. Production: Deployable assets

Interview Questions

1. Basic: npm aur yarn mein kya difference hai?

Answer: Dono package managers hain. Yarn faster hai, better caching, lockfile support. npm default hai Node.js ke saath.

2. Basic: Babel kya karta hai?

Answer: Modern JavaScript (ES6+) ko older browsers ke liye compatible ES5 code mein transpile karta hai.

3. Intermediate: Webpack aur Vite mein main difference kya hai?

Answer: Webpack mature bundler hai, Vite faster dev server with native ES modules. Vite development mein fast, Webpack production mein mature.

4. Intermediate: Tree shaking kya hai?

Answer: Unused code ko bundle se remove karna. Dead code elimination for smaller bundle sizes.

5. Advanced: Hot Module Replacement (HMR) kaise work karta hai?

Answer: Development mein code changes ko page refresh ke bina apply karta hai. WebSocket connection se browser aur dev server communicate karte hain.
