JavaScript mein OOP ko samajhte hain - Constructor Functions, ES6 Classes, aur Inheritance

üìö OOP Kya Hai?

Object-Oriented Programming (OOP) ek programming paradigm hai jahan hum real-world entities ko objects ke form mein represent karte hain. JavaScript mein OOP ko implement karne ke liye humein multiple approaches milte hain.

Socho agar aapko ek car showroom ka software banana hai - har car ek object hogi jisme uski properties (color, model, price) aur methods (start, stop, accelerate) honge. OOP se hum isse efficiently organize kar sakte hain!

Key Concepts:

Concept Explanation
Encapsulation Data aur methods ko ek unit (object) mein bundle karna
Inheritance Ek class apni properties aur methods doosri class ko de sakti hai
Polymorphism Same method ka different implementations
Abstraction Complex details ko hide karke sirf essential features dikhana

üîß Constructor Functions

ES6 classes se pehle, JavaScript mein constructor functions use karte the objects create karne ke liye. Yeh functions capital letter se start hote hain aur new keyword ke saath use hote hain.

// Constructor Function Example function Person(name, age) { this.name = name; this.age = age; this.greet = function() { return `Namaste! Mera naam ${this.name} hai.`; }; } // Object Creation const person1 = new Person('Rahul', 25); console.log(person1.greet());

üéì ES6 Classes

ES6 mein classes introduce hui jo OOP ko aur bhi cleaner aur easier bana deti hain. Yeh internally constructor functions ki tarah hi kaam karti hain, lekin syntax bahut better hai!

// Basic ES6 Class class Student { constructor(name, rollNo) { this.name = name; this.rollNo = rollNo; } study() { return `${this.name} padh raha hai`; } } const student1 = new Student('Priya', 101);
// Instance Methods vs Static Methods class Calculator { constructor(value) { this.value = value; } // Instance Method - har object ke liye alag add(num) { this.value += num; return this.value; } // Static Method - class ke saath directly use hota hai static multiply(a, b) { return a * b; } } Calculator.multiply(5, 3); // 15
// Getters and Setters class BankAccount { constructor(owner) { this.owner = owner; this._balance = 0; // private property } // Getter - property ki tarah access kar sakte hain get balance() { return `‚Çπ${this._balance}`; } // Setter - value set karne ke liye validation set balance(amount) { if(amount < 0) { console.log('Negative amount nahi ho sakta!'); } else { this._balance = amount; } } }
// Static Methods - Utility Functions class MathHelper { static square(num) { return num * num; } static cube(num) { return num * num * num; } static max(...numbers) { return Math.max(...numbers); } } // Static methods ko class ke saath directly call karte hain MathHelper.square(5); // 25 MathHelper.max(10, 20, 30); // 30

üîó Inheritance (Virasat)

Inheritance ka matlab hai ki ek class (child) doosri class (parent) ki properties aur methods ko inherit kar sakti hai. Isse code reusability badh jaati hai!

Jaise - Vehicle ek parent class hai, aur Car aur Bike uski child classes hain. Dono Vehicle ki common properties inherit kar lenge.

// Parent Class class Vehicle { constructor(brand, year) { this.brand = brand; this.year = year; this.isRunning = false; } start() { this.isRunning = true; return `${this.brand} start ho gayi!`; } stop() { this.isRunning = false; return `${this.brand} band ho gayi!`; } } // Child Class - Car class Car extends Vehicle { constructor(brand, year, doors) { super(brand, year); // parent constructor ko call karta hai this.doors = doors; } honk() { return 'Beep Beep! üöó'; } } // Child Class - Bike class Bike extends Vehicle { constructor(brand, year, type) { super(brand, year); this.type = type; // sports, cruiser, etc } wheelie() { return 'Bike ek wheel pe chal rahi hai! üèçÔ∏è'; } }

üéØ Complete Example: Bank Account System

Ab sab concepts ko combine karke ek practical example dekhte hain - Bank Account System jisme inheritance, getters/setters, aur static methods ka use hai.

class BankAccount { static accountCount = 0; constructor(owner, initialBalance = 0) { this.owner = owner; this._balance = initialBalance; this.accountNumber = ++BankAccount.accountCount; } get balance() { return `‚Çπ${this._balance}`; } deposit(amount) { if(amount > 0) { this._balance += amount; return `‚Çπ${amount} deposit ho gaye!`; } } withdraw(amount) { if(amount > this._balance) { return 'Insufficient balance!'; } this._balance -= amount; return `‚Çπ${amount} withdraw ho gaye!`; } static getTotalAccounts() { return `Total accounts: ${BankAccount.accountCount}`; } } class SavingsAccount extends BankAccount { constructor(owner, initialBalance, interestRate) { super(owner, initialBalance); this.interestRate = interestRate; } addInterest() { const interest = this._balance * (this.interestRate / 100); this._balance += interest; return `‚Çπ${interest.toFixed(2)} interest add hua!`; } }

üíº Interview Questions & Answers

Q1. Constructor function aur ES6 class mein kya difference hai? Easy
Answer:

Dono internally same kaam karte hain lekin syntax aur behavior mein differences hain:

üîπ Constructor Function:
‚Ä¢ Regular function hai jo capital letter se start hota hai
‚Ä¢ Methods ko prototype mein manually add karna padta hai
‚Ä¢ 'new' keyword mandatory hai object create karne ke liye
‚Ä¢ Hoisted hote hain (function declarations ki tarah)

Example:
function Person(name) {
  this.name = name;
}
Person.prototype.greet = function() {
  console.log('Hello ' + this.name);
};
üîπ ES6 Class:
‚Ä¢ Cleaner aur readable syntax
‚Ä¢ Methods automatically prototype mein add hote hain
‚Ä¢ 'extends' keyword se inheritance easy ho jaati hai
‚Ä¢ Hoisting nahi hoti (Temporal Dead Zone mein rehte hain)
‚Ä¢ Strict mode mein automatically run hoti hai
‚Ä¢ 'new' bina use kiye error throw karti hai

Example:
class Person {
  constructor(name) {
    this.name = name;
  }
  greet() {
    console.log('Hello ' + this.name);
  }
}
üí° Key Difference: Classes syntactic sugar hain lekin behavior mein differences hain jaise hoisting aur strict mode.
Q2. 'this' keyword class ke andar kaise kaam karta hai? Easy
Answer:

Class ke andar 'this' keyword current instance (object) ko refer karta hai jo us class se create hua hai.

üî∏ Constructor mein 'this':
Constructor mein 'this' naye object ko point karta hai jo abhi create ho raha hai. Jab hum 'new ClassName()' likhte hain, JavaScript internally ek empty object banata hai aur 'this' us object ko refer karta hai.

class Car {
  constructor(brand) {
    this.brand = brand; // 'this' = new car object
  }
}
const car1 = new Car('Honda'); // this = car1
const car2 = new Car('Toyota'); // this = car2
üî∏ Methods mein 'this':
Instance methods mein 'this' us object ko refer karta hai jisne method ko call kiya hai.

‚ö†Ô∏è Important Points:
‚Ä¢ Arrow functions apna 'this' nahi rakhte, parent scope ka 'this' use karte hain
‚Ä¢ Method ko variable mein assign karke call karne par 'this' context lost ho sakta hai
‚Ä¢ .bind(), .call(), .apply() se 'this' ko explicitly set kar sakte hain

Example of Lost Context:
const obj = { name: 'Test', getName() { return this.name; } };
const fn = obj.getName;
fn(); // undefined - 'this' context lost!
fn.call(obj); // 'Test' - context restored
Q3. Static methods kya hote hain aur kab use karte hain? Medium
Answer:

Static methods class ke saath directly bind hote hain, individual instances (objects) ke saath nahi. Inhe call karne ke liye object banana zaruri nahi hai.

üîπ Key Characteristics:
‚Ä¢ ClassName.methodName() se directly call hote hain
‚Ä¢ 'this' keyword class constructor ko refer karta hai, instance ko nahi
‚Ä¢ Instance se access nahi kar sakte (obj.staticMethod() ‚ùå)
‚Ä¢ Memory efficient - sirf ek copy memory mein rahti hai

üìå Kab Use Karte Hain:

1. Utility/Helper Functions:
class MathUtils {
  static add(a, b) { return a + b; }
  static square(n) { return n * n; }
}
MathUtils.add(5, 3); // 8
2. Factory Methods (Objects create karne ke liye):
class User {
  static createAdmin(name) {
    return new User(name, 'admin');
  }
}
3. Constants ya Configuration:
class Database {
  static MAX_CONNECTIONS = 100;
  static getConfig() { return {...}; }
}
4. Counter/Statistics:
class Product {
  static totalProducts = 0;
  constructor() { Product.totalProducts++; }
}
üí° Real Examples: Math.max(), Array.from(), Object.keys(), Date.now() - yeh sab static methods hain!
Q4. Getters aur Setters ki kya zarurat hai? Direct properties kyu nahi use karte? Medium
Answer:

Getters aur Setters data ko control aur protect karne ke liye use hote hain. Direct properties use karne se data validation aur encapsulation ka problem hota hai.

üî∏ Getters aur Setters Kyu Zaruri Hain:

1. Data Validation:
Setter mein value set karne se pehle validation kar sakte hain:
class Person {
  set age(value) {
    if(value < 0 || value > 150) {
      throw new Error('Invalid age!');
    }
    this._age = value;
  }
}
2. Computed Properties:
Getter se dynamic values calculate kar sakte hain:
class Rectangle {
  constructor(width, height) {
    this.width = width;
    this.height = height;
  }
  get area() {
    return this.width * this.height; // har baar calculate
  }
}
3. Encapsulation (Data Protection):
Internal representation ko hide kar sakte hain:
class BankAccount {
  #balance = 0; // private
  get balance() {
    return `‚Çπ${this.#balance}`; // formatted output
  }
}
4. Backwards Compatibility:
Implementation change karne par bhi API same rahta hai. Direct property ko getter/setter mein convert kar sakte hain bina code break kiye.

5. Side Effects:
Value set/get karte waqt additional actions perform kar sakte hain (logging, notifications, etc.)

‚ö†Ô∏è Direct Property Problem:
person.age = -5; // ‚ùå Invalid age set ho jayegi!
person.age = 'twenty'; // ‚ùå String bhi set ho jayegi!
Q5. 'super' keyword ka kya use hai aur kab zaruri hai? Medium
Answer:

'super' keyword child class se parent class ko access karne ke liye use hota hai. Inheritance mein yeh bahut important hai.

üîπ super() - Parent Constructor Call:
Child class mein constructor hai toh super() call karna MANDATORY hai, woh bhi 'this' use karne se pehle.

class Animal {
  constructor(name) {
    this.name = name;
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // ‚úÖ Parent constructor call - ZARURI!
    this.breed = breed; // 'this' sirf super() ke baad use kar sakte hain
  }
}
üîπ super.method() - Parent Method Call:
Parent class ke methods ko call karne ke liye, especially jab woh method override ho gaya ho.

class Vehicle {
  start() {
    return 'Vehicle started';
  }
}

class Car extends Vehicle {
  start() {
    const msg = super.start(); // Parent ka method call
    return msg + ' and engine is warming up!';
  }
}
‚ö†Ô∏è Important Rules:
‚Ä¢ Child class mein agar constructor hai toh super() zaruri hai
‚Ä¢ super() ko 'this' keyword use karne se PEHLE call karna padta hai
‚Ä¢ super() sirf constructor ke andar call hota hai
‚Ä¢ super.methodName() kisi bhi method mein use kar sakte hain

‚ùå Common Mistakes:
constructor(name) {
  this.name = name; // ‚ùå Error! 'this' before super()
  super(name);
}

constructor(name) {
  // ‚ùå Error! super() missing
  this.name = name;
}
Q6. Private properties (#property) kya hain aur kaise kaam karti hain? Hard
Answer:

Private properties ES2022 mein introduce hui - '#' symbol se start hoti hain aur class ke bahar se bilkul bhi accessible nahi hoti. Yeh true encapsulation provide karti hain.

üî∏ Syntax aur Declaration:
class User {
  #password; // Private field declaration
  #loginAttempts = 0; // With default value

  constructor(username, password) {
    this.username = username; // Public
    this.#password = password; // Private
  }

  validatePassword(pass) {
    return this.#password === pass; // ‚úÖ Class ke andar accessible
  }
}

const user = new User('rahul', 'secret123');
console.log(user.username); // ‚úÖ 'rahul'
console.log(user.#password); // ‚ùå SyntaxError!
üî∏ Key Characteristics:
‚Ä¢ Class body mein declare hona chahiye (constructor se pehle)
‚Ä¢ Class ke methods mein accessible hain
‚Ä¢ Child classes mein bhi accessible NAHI hain
‚Ä¢ Truly private - koi bhi external code access nahi kar sakta
‚Ä¢ Private methods bhi ban sakte hain: #privateMethod() {}

üî∏ _property vs #property:
Convention (_property) Private (#property)
Sirf naming convention hai Language-level privacy
Actually accessible hai Bilkul accessible nahi
obj._prop se access ho jayega obj.#prop SyntaxError throw karega
Child classes inherit kar sakti hain Child classes access nahi kar sakti

üí° Use Case Example:
class BankAccount {
  #balance = 0;
  #pin;

  constructor(pin) {
    this.#pin = pin;
  }

  withdraw(amount, inputPin) {
    if(this.#pin !== inputPin) return 'Wrong PIN!';
    if(this.#balance >= amount) {
      this.#balance -= amount;
    }
  }
}
// PIN aur balance ko directly access nahi kar sakte!
Q7. Method Overriding kya hai? Real-world example ke saath explain karo. Hard
Answer:

Method Overriding tab hoti hai jab child class parent class ke method ko same name se redefine karta hai. Isse child class apna specific behavior implement kar sakti hai.

üî∏ Basic Concept:
class Animal {
  makeSound() {
    return "Some generic sound";
  }
  eat() {
    return "Animal is eating";
  }
}

class Dog extends Animal {
  makeSound() { // ‚úÖ Override
    return "Bhow Bhow! üêï";
  }
  // eat() override nahi kiya - parent ka use hoga
}

class Cat extends Animal {
  makeSound() { // ‚úÖ Override
    return "Meow Meow! üêà";
  }
}

const dog = new Dog();
const cat = new Cat();
dog.makeSound(); // "Bhow Bhow! üêï"
cat.makeSound(); // "Meow Meow! üêà"
dog.eat(); // "Animal is eating" - parent ka method
üî∏ Real-World Example - Payment System:
class Payment {
  processPayment(amount) {
    return `Processing ‚Çπ${amount}`;
  }
}

class CreditCardPayment extends Payment {
  processPayment(amount) {
    // Credit card specific logic
    const fee = amount * 0.02; // 2% fee
    return `Credit Card: ‚Çπ${amount + fee} (with fee)`;
  }
}

class UPIPayment extends Payment {
  processPayment(amount) {
    // UPI specific logic
    return `UPI: ‚Çπ${amount} (instant, no fee)`;
  }
}

class CashPayment extends Payment {
  processPayment(amount) {
    return `Cash: ‚Çπ${amount} received`;
  }
}
üî∏ Benefits of Method Overriding:
‚Ä¢ Polymorphism: Same interface, different implementations
‚Ä¢ Flexibility: Har child class apna behavior define kar sakti hai
‚Ä¢ Code Reusability: Common code parent mein, specific code child mein
‚Ä¢ Maintainability: Changes ek jagah karne se sab jagah reflect hote hain

üî∏ super.method() ke saath:
Parent ka method bhi use kar sakte hain aur uske saath extra functionality add kar sakte hain:
class Employee {
  getDetails() {
    return "Employee details";
  }
}

class Manager extends Employee {
  getDetails() {
    const baseDetails = super.getDetails(); // Parent call
    return baseDetails + " + Team management";
  }
}
üí° Real Interview Scenario:
Interviewer: "Agar 100 types ke animals hain aur sabka sound alag hai, kaise handle karoge?"
Answer: "Method overriding use karunga - base Animal class mein makeSound() method rahega, aur har animal type (Dog, Cat, Lion, etc.) us method ko override karke apna specific sound return karega. Isse code maintainable aur scalable rahega."
