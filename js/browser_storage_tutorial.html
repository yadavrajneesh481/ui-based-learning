Cookies, LocalStorage, SessionStorage & IndexedDB

üìö Introduction - Storage Kya Hai?

Browser storage ka matlab hai apne web browser mein data save karna. Jab aap koi website visit karte ho, wo website aapke browser mein kuch information store kar sakti hai, taaki next time jab aap visit karo tab wo information use ho sake. Isse user experience better hota hai!

Browser Storage allows websites to store data locally on the user's device. This enables features like remembering user preferences, caching data, and maintaining session state even when offline.

‚öñÔ∏è Storage Types Comparison

Teen main types hai browser storage ke: Cookies, LocalStorage, aur SessionStorage. Sabke apne advantages aur use cases hain. Chaliye inhe compare karte hain!

üç™ Cookies

Purana tarika data store karne ka, server ke saath share hota hai

  • Capacity: ~4KB per cookie
  • Expiration: Set kar sakte ho
  • Accessibility: Server + Client side
  • Storage Location: Browser & Server
  • Sent with Requests: Yes

üíæ LocalStorage

Permanent storage, browser close karne ke baad bhi rahta hai

  • Capacity: ~5-10MB
  • Expiration: Never (manually clear)
  • Accessibility: Client side only
  • Storage Location: Browser only
  • Sent with Requests: No

‚è±Ô∏è SessionStorage

Tab band hone tak hi rahta hai data

  • Capacity: ~5-10MB
  • Expiration: Tab close hone par
  • Accessibility: Client side only
  • Storage Location: Browser only
  • Sent with Requests: No

üóÉÔ∏è IndexedDB - Advanced Storage

IndexedDB bahut powerful hai! Ye ek complete database hai browser mein. Large amounts of data store kar sakte ho, complex queries run kar sakte ho. Offline applications ke liye best hai!

IndexedDB Features:

  • Capacity: 50MB+ (depends on device)
  • Data Types: Objects, Arrays, Blobs, Files
  • Queries: Index-based searching
  • Transactions: ACID compliant
  • Asynchronous: Non-blocking operations

Try IndexedDB

Output yaha dikhega...
Basic IndexedDB Example: 
// Database open karo
const request = indexedDB.open('MyDatabase', 1);

request.onsuccess = (event) => {
    const db = event.target.result;
    console.log('Database opened successfully');
};

// Object store create karo
request.onupgradeneeded = (event) => {
    const db = event.target.result;
    const store = db.createObjectStore('users', { keyPath: 'id' });
    store.createIndex('name', 'name', { unique: false });
};

Note: IndexedDB complex hai, production mein libraries like Dexie.js use karte hain easier development ke liye!

üé® Practice: Dark Mode Preference Saver

Is page ka dark mode already localStorage use kar raha hai! Top-right corner mein button click karo aur dekho kaise preference save hoti hai. Page reload karne ke baad bhi same theme rahega!

How It Works:

  1. User theme button click karta hai
  2. JavaScript localStorage mein preference save kar deta hai
  3. Page load hone par localStorage check hota hai
  4. Saved preference ke according theme apply ho jata hai
Implementation Code: 
// Theme toggle function
function toggleTheme() {
    document.body.classList.toggle('dark-mode');
    const isDark = document.body.classList.contains('dark-mode');
    
    // LocalStorage mein save karo
    localStorage.setItem('theme', isDark ? 'dark' : 'light');
    
    // Button text update karo
    const button = document.getElementById('themeToggle');
    button.textContent = isDark ? '‚òÄÔ∏è Light Mode' : 'üåô Dark Mode';
}

// Page load par theme apply karo
window.addEventListener('load', () => {
    const savedTheme = localStorage.getItem('theme');
    if (savedTheme === 'dark') {
        document.body.classList.add('dark-mode');
        document.getElementById('themeToggle').textContent = '‚òÄÔ∏è Light Mode';
    }
});

üíº Interview Questions

Yaha 7 important interview questions hain browser storage ke baare mein:

Q1: LocalStorage aur SessionStorage mein kya difference hai?

Answer:

LocalStorage: Data permanently store hota hai until explicitly deleted. Browser close karne ke baad bhi data rahta hai.

SessionStorage: Data tab/window close hone tak hi rahta hai. Session end hone par automatically delete ho jata hai.

Use Cases:

  • LocalStorage: User preferences, settings, cached data
  • SessionStorage: Form data, temporary cart items, wizard steps

Q2: Cookies ko localStorage se better kab use karte hain?

Answer:

Cookies use karo jab:

  • Server-side access chahiye: Cookies automatically har HTTP request ke saath server ko bheje jate hain
  • Authentication: Session tokens aur login info store karne ke liye
  • Expiry control: Specific time ke baad data expire hona chahiye
  • Cross-subdomain sharing: Different subdomains ke beech data share karna ho

LocalStorage use karo jab server ko data bhejne ki zarurat nahi hai aur storage space zyada chahiye.

Q3: LocalStorage mein object kaise store karte hain?

Answer:

LocalStorage sirf strings store kar sakta hai. Objects store karne ke liye JSON.stringify() aur JSON.parse() use karo:

// Object ko store karo
const user = { name: 'Rahul', age: 25 };
localStorage.setItem('user', JSON.stringify(user));

// Object ko retrieve karo
const storedUser = JSON.parse(localStorage.getItem('user'));
console.log(storedUser.name); // 'Rahul'

Q4: Browser storage secure hai kya? Security concerns kya hain?

Answer:

Security Concerns:

  • XSS Attacks: JavaScript access kar sakta hai localStorage/sessionStorage, isliye sensitive data (passwords, tokens) store nahi karna chahiye
  • Not Encrypted: Data plain text mein store hota hai
  • No Expiration (localStorage): Data automatically delete nahi hota

Best Practices:

  • Sensitive data ke liye httpOnly cookies use karo
  • Important tokens ke liye secure, httpOnly flags set karo
  • Data ko encrypt karke store karo agar necessary ho
  • Regular cleanup karo purane data ka

Q5: IndexedDB localStorage se kab better hai?

Answer:

IndexedDB use karo jab:

  • Large Data: 50MB+ data store karna ho (localStorage sirf 5-10MB)
  • Complex Queries: Searching, filtering, indexing chahiye
  • Structured Data: Multiple tables/stores chahiye
  • Offline Applications: Complete offline database functionality chahiye
  • Binary Data: Files, images, blobs store karne ho
  • Transactions: ACID properties chahiye

Examples: Email clients, note-taking apps, offline games, PWAs

Q6: Storage quota kaise check karte hain aur kya hota hai jab full ho jata hai?

Answer:

Quota Check:

// Storage quota check karo
navigator.storage.estimate().then(estimate => {
    console.log('Used:', estimate.usage);
    console.log('Quota:', estimate.quota);
    console.log('Percentage:', (estimate.usage/estimate.quota)*100);
});

Jab storage full ho:

  • QuotaExceededError throw hota hai
  • Try-catch mein handle karo
  • Purane data delete karo (LRU strategy)
  • User ko notify karo
  • Compression use karo

Q7: Cache API aur localStorage mein kya difference hai?

Answer:

Cache API:

  • HTTP requests aur responses cache karta hai
  • Service Workers ke saath use hota hai
  • Offline functionality ke liye perfect
  • Network resources cache karta hai

LocalStorage:

  • Key-value pairs store karta hai
  • Simple data storage ke liye
  • Synchronous API
  • User preferences aur settings ke liye
