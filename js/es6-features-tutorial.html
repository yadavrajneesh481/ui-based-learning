Template Literals, Destructuring, Spread & Rest Operators

1. Template Literals & Multiline Strings

Kya hai Template Literals?

Template literals ek modern way hai strings ko define karne ka. Ye backticks (` `) use karte hain instead of quotes. Iska sabse bada benefit ye hai ki aap easily variables embed kar sakte ho aur multiline strings bana sakte ho without any hassle.

Technical Definition: Template literals are string literals allowing embedded expressions, denoted by backtick characters (`), and support multi-line strings and string interpolation with ${expression} syntax.

// Old way (ES5) var name = "Rahul"; var greeting = "Hello " + name + "!"; // New way (ES6+) - Template Literals const name = "Rahul"; const greeting = `Hello ${name}!`; // Multiline strings - Old way var multiline = "Line 1\n" + "Line 2\n" + "Line 3"; // Multiline strings - New way const multiline = `Line 1 Line 2 Line 3`;

2. Destructuring Arrays & Objects

Destructuring ka matlab?

Destructuring ek convenient way hai arrays aur objects se values ko extract karne ka aur unhe directly variables mein assign karne ka. Ye code ko cleaner aur more readable banata hai. Pehle hume har value ko individually access karna padta tha, ab ek line mein sab ho jata hai!

Technical Definition: Destructuring assignment is a syntax that unpacks values from arrays or properties from objects into distinct variables, providing a concise way to extract multiple values.

// Array Destructuring - Old way const colors = ['red', 'green', 'blue']; const firstColor = colors[0]; const secondColor = colors[1]; // Array Destructuring - New way const [firstColor, secondColor] = colors; // Object Destructuring - Old way const person = { name: 'Amit', age: 30 }; const name = person.name; const age = person.age; // Object Destructuring - New way const { name, age } = person;

3. Spread & Rest Operators (...)

Spread aur Rest Operators kya hain?

Ye dono operators same syntax use karte hain (three dots ...) lekin different contexts mein different kaam karte hain. Spread operator ek array ya object ko expand kar deta hai individual elements mein. Rest operator multiple elements ko ek single array mein collect kar leta hai.

Technical Definition: The spread operator (...) expands iterables into individual elements, while the rest operator (...) collects multiple elements into an array. Context determines which operation is performed.

// Spread Operator - Array copy const arr1 = [1, 2, 3]; const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5] // Spread Operator - Object merge const obj1 = { a: 1, b: 2 }; const obj2 = { ...obj1, c: 3 }; // {a: 1, b: 2, c: 3} // Rest Operator - Function parameters function sum(...numbers) { return numbers.reduce((a, b) => a + b); } sum(1, 2, 3, 4); // 10 // Rest in Destructuring const [first, ...rest] = [1, 2, 3, 4]; // first = 1, rest = [2, 3, 4]

üìù Interview Questions

Easy

Q1. What are template literals and how are they different from regular strings?

Answer: Template literals use backticks (`) instead of quotes and allow embedded expressions using ${expression} syntax. They support multi-line strings without escape characters and make string interpolation much cleaner. Example: `Hello ${name}!` instead of "Hello " + name + "!"
Easy

Q2. Explain array destructuring with an example.

Answer: Array destructuring allows unpacking values from arrays into separate variables. Example: const [a, b, c] = [1, 2, 3]; creates three variables a=1, b=2, c=3. You can also skip elements: const [first, , third] = [1, 2, 3]; gives first=1, third=3.
Medium

Q3. What's the difference between spread and rest operators? They use the same syntax (...).

Answer: Though they use the same syntax, context determines their behavior. Spread expands an array/object into individual elements (used in function calls, array literals). Rest collects multiple elements into an array (used in function parameters, destructuring). Example: const [first, ...rest] = [1,2,3] - here '...' is rest. const arr = [...oldArr, 4] - here '...' is spread.
Medium

Q4. How can you swap two variables using destructuring?

Answer: Using array destructuring: [a, b] = [b, a]; This creates a temporary array with swapped values and immediately destructures it back into the variables. No need for a temporary variable like in traditional swap: temp = a; a = b; b = temp;
Medium

Q5. Can you provide default values while destructuring? How?

Answer: Yes, you can provide default values. For arrays: const [a = 10, b = 20] = [5]; gives a=5, b=20. For objects: const {name = 'Guest', age = 18} = {name: 'John'}; gives name='John', age=18. This is useful when values might be undefined.
Hard

Q6. Explain nested destructuring with a real-world example.

Answer: Nested destructuring extracts values from nested objects/arrays. Example: const user = {name: 'Raj', address: {city: 'Delhi', pin: 110001}}; const {name, address: {city, pin}} = user; Now you have name='Raj', city='Delhi', pin=110001 as separate variables. Useful for API responses where data is deeply nested.
Hard

Q7. What happens when you use spread operator on an object with methods? Will the methods be copied?

Answer: Yes, spread creates a shallow copy, so methods are copied as references. However, 'this' context might break if the method references 'this'. Example: const obj1 = {name: 'Test', greet() {console.log(this.name)}}; const obj2 = {...obj1}; obj2.greet() works, but if you do: const {greet} = obj2; greet(); - 'this' will be undefined. Spread doesn't preserve prototypes or deep copy nested objects.
