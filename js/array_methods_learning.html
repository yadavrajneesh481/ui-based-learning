map() Transform

Definition: Creates a new array by transforming each element using a callback function.

Hinglish: Map method har element ko transform karke ek naya array banata hai. Original array change nahi hota.
Use case: Jab aapko har element par koi operation perform karke naya array chahiye.
const numbers = [1, 2, 3, 4, 5]; const doubled = numbers.map(num => num * 2); // Result: [2, 4, 6, 8, 10]

filter() Select

Definition: Creates a new array with elements that pass a test condition.

Hinglish: Filter method condition ke basis par elements ko select karta hai. Jo elements condition pass karte hain, wahi naye array mein aate hain.
const ages = [15, 22, 18, 30, 16]; const adults = ages.filter(age => age >= 18); // Result: [22, 18, 30]

reduce() Accumulate

Definition: Reduces an array to a single value by executing a reducer function.

Hinglish: Reduce sabse powerful method hai jo puri array ko ek single value mein convert kar deta hai. Ye accumulator use karta hai.
Parameters: accumulator (result store karta hai), currentValue, index, array
const prices = [100, 200, 300]; const total = prices.reduce((acc, price) => acc + price, 0); // Result: 600

find() Search

Definition: Returns the first element that satisfies the condition.

Hinglish: Find method pehla element return karta hai jo condition match karta hai. Agar koi nahi mila to undefined return hota hai.
const users = [{id: 1, name: 'Raj'}, {id: 2, name: 'Priya'}]; const user = users.find(u => u.id === 2); // Result: {id: 2, name: 'Priya'}

some() & every() Test

Definition: some() returns true if at least one element passes the test. every() returns true if all elements pass.

Hinglish:
‚Ä¢ some(): Agar koi bhi ek element condition pass kare to true
‚Ä¢ every(): Sabhi elements condition pass karein tabhi true

1 map() aur forEach() mein kya difference hai?
Answer:
‚Ä¢ map(): Naya array return karta hai with transformed values. Original array change nahi hota.
‚Ä¢ forEach(): Kuch return nahi karta, bas har element par callback execute karta hai. Side effects ke liye use hota hai.

Example:
const nums = [1,2,3];
const doubled = nums.map(n => n * 2); // [2,4,6]
nums.forEach(n => console.log(n)); // undefined return
2 reduce() method ki complexity kya hai aur iske real-world use cases?
Answer:
‚Ä¢ Time Complexity: O(n) - har element ko ek baar process karta hai
‚Ä¢ Real-world use cases:
1. Sum/Average calculation
2. Flattening arrays
3. Grouping objects by property
4. Counting occurrences
5. Object transformation

Example - Grouping:
const students = [{name: 'Raj', grade: 'A'}, {name: 'Priya', grade: 'A'}];
const grouped = students.reduce((acc, s) => {
  acc[s.grade] = acc[s.grade] || [];
  acc[s.grade].push(s.name);
  return acc;
}, {});
3 find() aur filter() mein difference explain karo with examples.
Answer:
‚Ä¢ find(): Sirf pehla matching element return karta hai ya undefined. Ek hi value chahiye to use karo.
‚Ä¢ filter(): Sabhi matching elements ka array return karta hai. Multiple values chahiye to use karo.

Example:
const users = [{id: 1, name: 'Raj', age: 25}, {id: 2, name: 'Priya', age: 30}];

// find() - single result
const user = users.find(u => u.age > 20); // {id: 1, name: 'Raj', age: 25}

// filter() - multiple results
const adults = users.filter(u => u.age > 20); // [{id: 1...}, {id: 2...}]
4 sort() method original array ko mutate karta hai. Isko immutable kaise banayein?
Answer:
Original array ko preserve karne ke liye pehle copy banana padega. 3 ways hain:

1. Spread Operator:
const original = [3, 1, 4, 2];
const sorted = [...original].sort((a,b) => a-b);

2. Array.from():
const sorted = Array.from(original).sort((a,b) => a-b);

3. slice():
const sorted = original.slice().sort((a,b) => a-b);

Ye best practice hai kyunki original data safe rehta hai.
5 Method chaining mein performance considerations kya hain?
Answer:
Performance Concerns:
‚Ä¢ Har method puri array ko traverse karta hai (O(n))
‚Ä¢ Multiple chains = multiple iterations
‚Ä¢ Memory overhead - har method naya array create karta hai

Optimization Tips:
1. Combine operations: Multiple filters ko ek mein merge karo
2. Order matters: Pehle filter karo (reduce size), phir map
3. Use reduce: Complex chains ko single reduce mein convert karo

Example:
// Bad - 3 iterations
arr.filter(x => x > 0).map(x => x * 2).filter(x => x < 100)

// Better - 2 iterations
arr.filter(x => x > 0 && x * 2 < 100).map(x => x * 2)
6 some() aur every() ka practical use case scenario batao.
Answer:
some() - "Koi bhi ek" validation:
‚Ä¢ Form validation: Koi bhi field empty hai?
‚Ä¢ Permission check: User ke paas koi bhi admin role hai?
‚Ä¢ Search: Array mein koi keyword present hai?

every() - "Sabhi" validation:
‚Ä¢ Form validation: Sabhi required fields filled hain?
‚Ä¢ Age verification: Sabhi users adult hain?
‚Ä¢ Quality check: Sabhi products stock mein hain?

Real Example:
const cart = [{name: 'Laptop', inStock: true}, {name: 'Mouse', inStock: false}];

// Check if any item out of stock
const hasOutOfStock = cart.some(item => !item.inStock); // true

// Check if all items in stock
const allInStock = cart.every(item => item.inStock); // false
7 Complex nested data ko flatten karne ke liye array methods kaise use karein?
Answer:
Nested arrays ko flatten karne ke multiple ways hain:

1. flat() method (ES2019):
const nested = [1, [2, 3], [4, [5, 6]]];
const flat = nested.flat(2); // [1, 2, 3, 4, 5, 6]

2. reduce() with recursion:
function flatten(arr) {
  return arr.reduce((acc, item) => {
    return acc.concat(Array.isArray(item) ? flatten(item) : item);
  }, []);
}

3. flatMap() for one level:
const users = [{id: 1, tags: ['js', 'react']}, {id: 2, tags: ['node']}];
const allTags = users.flatMap(u => u.tags); // ['js', 'react', 'node']
