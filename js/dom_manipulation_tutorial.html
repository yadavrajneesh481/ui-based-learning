üéØ Selecting Elements

JavaScript mein elements ko select karne ke liye mainly do methods use hote hain:

1. querySelector: CSS selector use karke kisi bhi element ko select karta hai. Yeh pehla matching element return karta hai.
2. getElementById: Element ko uske ID se select karta hai. Yeh bahut fast hai aur specific element ke liye best hai.

Bonus: querySelectorAll sabhi matching elements ko select karta hai (NodeList return karta hai).
// querySelector - CSS selector use karta hai
const element = document.querySelector('.myClass');
const button = document.querySelector('#myButton');

// getElementById - ID se select karta hai
const heading = document.getElementById('mainHeading');

// querySelectorAll - sabhi matching elements
const allButtons = document.querySelectorAll('button');

‚úèÔ∏è Modifying Content, Attributes & Styles

Jab aap element select kar lete ho, toh usko modify karna bahut easy hai:

Content: textContent aur innerHTML se text ya HTML change kar sakte hain.
Attributes: setAttribute() aur getAttribute() se attributes manage kar sakte hain.
Styles: style property se directly CSS properties change kar sakte hain.
// Content change karna
element.textContent = 'New Text';
element.innerHTML = '<strong>Bold Text</strong>';

// Attributes change karna
element.setAttribute('class', 'newClass');
element.getAttribute('id');

// Styles change karna
element.style.color = 'blue';
element.style.backgroundColor = '#f0f0f0';

‚ûï‚ûñ Creating & Removing Elements

JavaScript se aap dynamically elements create aur remove kar sakte ho:

Creating: createElement() se naya element banate hain, phir appendChild() ya append() se DOM mein add karte hain.
Removing: remove() ya removeChild() se elements ko DOM se hata sakte hain.
// Naya element create karna
const newDiv = document.createElement('div');
newDiv.textContent = 'I am new!';
newDiv.className = 'myClass';

// DOM mein add karna
document.body.appendChild(newDiv);

// Element remove karna
newDiv.remove();

üé§ Interview Questions

Q1. What is the DOM and why is it important?

Answer: DOM (Document Object Model) ek programming interface hai jo HTML aur XML documents ko represent karta hai. Yeh ek tree structure hai jahan har element, attribute, aur text ek node hota hai.

Importance:
‚Ä¢ JavaScript se HTML ko dynamically manipulate kar sakte hain
‚Ä¢ User interactions handle kar sakte hain
‚Ä¢ Content ko runtime mein change kar sakte hain
‚Ä¢ Dynamic aur interactive web applications bana sakte hain

Q2. querySelector vs getElementById - Kaunsa fast hai aur kyun?

Answer: getElementById querySelector se fast hai.

Reason:
‚Ä¢ getElementById directly ID-based lookup karta hai (O(1) complexity)
‚Ä¢ querySelector ko CSS selector parse karna padta hai, phir search karna padta hai
‚Ä¢ getElementById only IDs ke liye kaam karta hai (limitation)
‚Ä¢ querySelector zyada flexible hai - kisi bhi CSS selector use kar sakte ho

Best Practice: Agar sirf ID se select karna hai toh getElementById use karo, complex selectors ke liye querySelector.

Q3. textContent vs innerHTML mein kya difference hai?

Answer:

textContent:
‚Ä¢ Sirf plain text return/set karta hai
‚Ä¢ HTML tags ko text ki tarah treat karta hai
‚Ä¢ Fast aur secure hai
‚Ä¢ XSS attacks se safe hai

innerHTML:
‚Ä¢ HTML markup ko parse karta hai
‚Ä¢ HTML tags ko render karta hai
‚Ä¢ Slower hai kyunki parsing hoti hai
‚Ä¢ Security risk hai agar user input directly use kiya

Example: element.innerHTML = '<b>Bold</b>' ‚Üí Bold text render hoga
element.textContent = '<b>Bold</b>' ‚Üí <b>Bold</b> as text dikhega

Q4. appendChild vs append mein kya difference hai?

Answer:

appendChild:
‚Ä¢ Purana method hai (DOM Level 1)
‚Ä¢ Sirf ek Node object add kar sakta hai
‚Ä¢ Added node ko return karta hai
‚Ä¢ String directly pass nahi kar sakte

append:
‚Ä¢ Naya method hai (DOM Level 4)
‚Ä¢ Multiple nodes ya strings add kar sakte hain
‚Ä¢ Kuch return nahi karta (undefined)
‚Ä¢ String directly pass kar sakte ho

Example:
parent.appendChild(child) ‚Üí works
parent.append(child1, child2, 'text') ‚Üí works
parent.appendChild('text') ‚Üí error!

Q5. How can you prevent XSS attacks when manipulating DOM?

Answer: XSS (Cross-Site Scripting) attacks ko prevent karne ke liye:

Best Practices:
‚Ä¢ textContent use karo innerHTML ki jagah (jab possible ho)
‚Ä¢ User input ko sanitize karo before adding to DOM
‚Ä¢ Never use eval() with user input
‚Ä¢ Use setAttribute() instead of directly setting HTML attributes
‚Ä¢ Content Security Policy (CSP) headers implement karo
‚Ä¢ Modern frameworks use karo jo automatic escaping provide karte hain

Example:
‚ùå element.innerHTML = userInput; // Dangerous!
‚úÖ element.textContent = userInput; // Safe

Q6. Explain Event Delegation in context of DOM manipulation

Answer: Event Delegation ek technique hai jahan aap parent element pe event listener lagate ho, child elements pe individually listeners lagane ki bajaye.

Advantages:
‚Ä¢ Memory efficient - kam event listeners
‚Ä¢ Dynamic elements automatically handle ho jate hain
‚Ä¢ Better performance for large lists

How it works: Events "bubble up" karte hain DOM tree mein, toh parent pe listener lagake child events catch kar sakte ho using event.target.

Example:
parent.addEventListener('click', (e) => {
  if(e.target.matches('.delete-btn')) {
    // Handle delete
  }
});

Q7. What are Virtual DOM and how does it relate to actual DOM?

Answer: Virtual DOM ek lightweight JavaScript representation hai actual DOM ka.

Key Points:
‚Ä¢ React aur Vue jaise frameworks Virtual DOM use karte hain
‚Ä¢ Changes pehle Virtual DOM mein hote hain (fast)
‚Ä¢ Phir diffing algorithm se actual DOM mein minimal changes apply hoti hain
‚Ä¢ Direct DOM manipulation se zyada efficient hai

Why it matters:
‚Ä¢ DOM manipulation expensive hai (slow)
‚Ä¢ Virtual DOM batching aur optimization allow karta hai
‚Ä¢ Better performance in complex applications

Process:
1. State change hota hai
2. New Virtual DOM create hota hai
3. Old aur New Virtual DOM compare hote hain
4. Sirf differences actual DOM mein update hoti hain
