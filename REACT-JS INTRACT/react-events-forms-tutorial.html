1. Synthetic Events

Synthetic Events kya hain? React apne khud ke event system use karta hai jo browser ke native events ko wrap karta hai. Isko SyntheticEvent kehte hain.

Kyun use karte hain? Cross-browser compatibility ke liye. Sabhi browsers mein same behavior milta hai.

Interactive Demo: Event Properties
// Synthetic Event Example function handleClick(e) { e.preventDefault(); // Works across all browsers console.log(e.type); // 'click' console.log(e.target); // Button element console.log(e.currentTarget); // Current element }

Important Properties

Property Description
e.target Jis element pe click hua woh (actual element)
e.currentTarget Jis element pe event listener laga hai woh
e.preventDefault() Default behavior ko rok deta hai
e.stopPropagation() Event bubbling ko rok deta hai

Form Validation zaroori hai taaki user correct data enter kare. Validation teen types ka hota hai:

  • Client-side validation: Browser mein hi check karo
  • Real-time validation: Har keystroke pe validate karo
  • Submit-time validation: Form submit hone pe check karo
Interactive Form with Validation
// Validation Function Example const validateForm = (values) => { const errors = {}; if (!values.name || values.name.length < 3) { errors.name = 'Name must be at least 3 characters'; } if (!values.email || !/\S+@\S+\.\S+/.test(values.email)) { errors.email = 'Valid email required'; } return errors; }

Q1: What are Synthetic Events in React? Why are they used? Easy

Answer: Synthetic Events React ke cross-browser wrapper hain native browser events ke around. React apna khud ka event system use karta hai jo consistent behavior provide karta hai sabhi browsers mein.

Key Points:
  • Cross-browser compatibility ensure karta hai
  • Event pooling ke through performance optimize karta hai
  • Same API milta hai har browser mein
  • Native event ko wrap karke consistent interface deta hai

Q2: Controlled vs Uncontrolled Components mein kya difference hai? Medium

Answer:

Controlled Components:
  • React state input ki value ko control karta hai
  • Har change pe state update hota hai
  • Real-time validation easy hai
  • Example: <input value={state} onChange={handler} />
Uncontrolled Components:
  • DOM khud data manage karta hai
  • Ref use karke value access karte hain
  • Simple forms ke liye suitable
  • Example: <input ref={inputRef} />

Q3: How do you handle multiple inputs in a single onChange handler? Medium

Answer: Ek single object state use karte hain aur computed property name use karke dynamic key update karte hain:

const [formData, setFormData] = useState({name: '', email: ''});

const handleChange = (e) => {
  const {name, value} = e.target;
  setFormData(prev => ({...prev, [name]: value}));
};

<input name="name" onChange={handleChange} />
<input name="email" onChange={handleChange} />
Benefits: Less code, scalable, aur easy to maintain.

Q4: What is event bubbling and how do you stop it in React? Medium

Answer: Event bubbling ek mechanism hai jismein events child element se parent element tak propagate hote hain. Jab aap child pe click karte ho, pehle child ka handler chalega, phir parent ka.

Stop karne ke tarike:
  • e.stopPropagation() - Event ko parent tak jane se rokta hai
  • e.preventDefault() - Default browser behavior rokta hai
const handleClick = (e) => {
  e.stopPropagation(); // Parent handler nahi chalega
  console.log('Only child handler runs');
};

Q6: How do you validate forms in React? Explain different approaches. Hard

Answer: React mein form validation ke multiple approaches hain:

1. Manual Validation:
  • Custom validation functions likhna
  • onChange ya onSubmit pe validate karna
  • Full control milta hai but boilerplate code zyada
2. HTML5 Validation:
  • Built-in attributes: required, pattern, min, max
  • Quick implementation but limited customization
3. Schema-based Validation (Yup):
  • Declarative validation rules
  • Reusable schemas
  • Complex validations easily possible
4. Real-time vs Submit-time:
  • Real-time: Har keystroke pe validate (better UX)
  • Submit-time: Form submit pe validate (better performance)
  • On Blur: Field leave karne pe validate (balanced approach)
Best Practice: Formik + Yup combination sabse popular hai kyunki yeh clean, maintainable aur powerful hai.

Q7: What are the best practices for handling large forms with many inputs? Hard

Answer: Large forms ke liye efficient strategies:

1. Single State Object:
  • Saare form values ek object mein store karo
  • Generic onChange handler use karo with computed property names
2. Form Breakdown:
  • Large form ko smaller sections mein divide karo
  • Multi-step forms banao with progress indicator
  • Lazy loading for better initial load time
3. Reusable Components:
  • Custom input components banao (FormInput, FormSelect, etc.)
  • Validation logic ko centralize karo
4. Performance Optimization:
  • Debouncing for expensive validations
  • React.memo() for input components
  • useCallback for event handlers
  • Field-level validation instead of form-level
5. State Management:
  • Complex forms ke liye Context API ya Redux use karo
  • Form state ko global banana for persistence
6. Libraries:
  • Formik ya React Hook Form for complex scenarios
  • Yup for schema validation
