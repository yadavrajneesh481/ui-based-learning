üåê SPA vs MPA

üìÑ SPA (Single Page Application)

Definition: A web application that loads a single HTML page and dynamically updates content.


Explanation: Ek hi page load hota hai initially, phir JavaScript dynamically content update karta hai. Page reload nahi hota!

  • ‚úÖ Fast navigation
  • ‚úÖ Better UX
  • ‚úÖ Less server load
  • ‚ùå Initial load time thoda zyada
  • ‚ùå SEO challenges

üìë MPA (Multi Page Application)

Definition: A web application with multiple HTML pages that reload on navigation.


Explanation: Har page ke liye alag HTML file. Jab navigate karo, pura page reload hota hai.

  • ‚úÖ Better SEO naturally
  • ‚úÖ Simpler architecture
  • ‚úÖ Easier to start
  • ‚ùå Slower navigation
  • ‚ùå More server requests

üîÑ Kaise Kaam Karta Hai?

  1. Step 1: React Virtual DOM ka snapshot banata hai (memory mein)
  2. Step 2: Jab state change hoti hai, naya Virtual DOM banta hai
  3. Step 3: React dono ko compare karta hai (Diffing Algorithm)
  4. Step 4: Sirf changes ko Real DOM mein update karta hai (Reconciliation)

üßµ React Fiber Architecture

React Fiber React ki reimplementation hai jo React 16 mein introduce hui. Ye reconciliation algorithm ka naya implementation hai.

üéØ Fiber Kya Solve Karta Hai?

Problem: Purane React mein, agar koi bada update ho raha ho, to browser freeze ho jata tha. User interactions (clicks, typing) block ho jate the.


Solution: Fiber kaam ko chhote pieces mein tod deta hai aur priority ke according karta hai.

‚ú® Key Features:

  • Incremental Rendering: Kaam ko pause kar sakte ho aur baad mein resume
  • Priority-based Updates: Important updates pehle (animations > data fetching)
  • Concurrency: Multiple tasks ek saath prepare ho sakte hain
  • Error Boundaries: Better error handling

üîç Behind The Scenes - React Kaise Render Karta Hai?

üìù Rendering Process (Step by Step):

  1. Trigger: Initial render ya state/props change
  2. Render Phase: React components ko call karta hai aur Virtual DOM tree banata hai
  3. Reconciliation: Purane aur naye Virtual DOM ko compare karta hai
  4. Commit Phase: Changes ko Real DOM mein apply karta hai
  5. Browser Paint: Browser screen ko update karta hai

Q1. React kya hai aur iske main features kya hain? Easy

Answer: React ek JavaScript library hai jo user interfaces banane ke liye use hoti hai. Main features:

  • Component-Based: UI ko reusable pieces mein divide karte hain
  • Virtual DOM: Efficient updates ke liye
  • Declarative: UI ki final state describe karte hain, React baaki sambhal leta hai
  • JSX: HTML-like syntax JavaScript mein
  • One-way Data Flow: Data parent se child ki taraf flow hota hai
Q2. Virtual DOM kya hai aur ye Real DOM se fast kyun hai? Easy

Answer: Virtual DOM React ka lightweight JavaScript representation hai Real DOM ka. Ye fast hai kyunki:

  • Ye memory mein rehta hai, browser paint nahi karta
  • React diffing algorithm se sirf actual changes find karta hai
  • Batch updates - multiple changes ko ek saath apply karta hai
  • Real DOM manipulation expensive hai (reflow, repaint), Virtual DOM mein ye nahi hota
  • Efficient reconciliation process se minimum DOM updates hote hain
Q3. SPA aur MPA mein kya difference hai? React kis type ka application hai? Medium

Answer:

SPA (Single Page Application): Ek hi HTML page load hota hai, content dynamically update hota hai JavaScript se. Page reload nahi hota. Better UX but SEO challenges.

MPA (Multi Page Application): Multiple HTML pages hain, har navigation pe new page load hota hai. Better SEO but slower navigation.

React primarily SPAs banane ke liye use hota hai, lekin Next.js jaise frameworks ke saath Server-Side Rendering bhi kar sakte hain jo SPA aur MPA ke benefits combine karta hai.

Q4. React Fiber kya hai aur isko introduce kyun kiya gaya? Medium

Answer: React Fiber React 16 mein introduce hua reimplementation hai reconciliation algorithm ka. Main goals:

  • Incremental Rendering: Rendering work ko chunks mein split kar sakte hain aur pause/resume kar sakte hain
  • Priority-based Updates: Different updates ko different priority assign kar sakte hain (animations high priority, data fetching low)
  • Concurrency: Multiple versions of UI ko parallel prepare kar sakte hain
  • Better Error Handling: Error boundaries se graceful error handling

Purane version mein long-running updates browser ko block kar dete the. Fiber ne ye problem solve kari.

Q5. React 18 ke main features kya hain? Concurrency kya hai? Hard

Answer: React 18 ke main features:

  • Automatic Batching: Promises, setTimeout mein bhi automatic batching
  • Transitions API: Non-urgent updates ko mark kar sakte hain using startTransition
  • Suspense for Data Fetching: Declarative loading states
  • New Root API: createRoot for concurrent features
  • Streaming SSR: Server-side rendering improvements

Concurrency: React ka ability multiple UI versions ko ek saath prepare karna. Main thread ko block nahi karta, urgent updates ko priority deta hai. Example: typing (urgent) aur search results update (non-urgent) ko alag handle karta hai.

Q6. Component-based architecture ke kya benefits hain? Explain with example. Medium

Answer: Component-based architecture ke benefits:

  • Reusability: Components ko multiple jagah use kar sakte hain
  • Maintainability: Isolated components ko maintain karna easy
  • Testability: Individual components ka testing simple
  • Scalability: Large applications ko organize karna easy
  • Separation of Concerns: Har component apni responsibility handle karta hai

Example: Ek Button component banao jo multiple forms mein reuse ho. Props se customization (color, size, text) control karo. Ek jagah change karo, sab jagah reflect ho.

Q7. React mein rendering process ko detail mein explain karo. Reconciliation kya hai? Hard

Answer: React Rendering Process:

  1. Trigger Phase: Initial mount ya setState/props change se render trigger hota hai
  2. Render Phase: React component functions ko call karta hai, JSX ko React elements mein convert karta hai, aur Virtual DOM tree bana
  3. Reconciliation Phase: Previous Virtual DOM aur new Virtual DOM ko compare karta hai (Diffing Algorithm). Ye phase pure hai aur side-effects nahi hote
  4. Commit Phase: Actual DOM mutations apply hote hain. useEffect/useLayoutEffect yahan run hote hain
  5. Browser Paint: Browser updated UI display karta hai

Reconciliation: Ye process hai jismein React efficiently determine karta hai ki DOM mein kya changes chahiye. React keys aur element types use karke optimize karta hai. Fiber architecture isko interruptible banata hai.
