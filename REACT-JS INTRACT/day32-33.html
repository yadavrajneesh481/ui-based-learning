üìö React Router v6 Kya Hai?

React Router ek powerful library hai jo aapke React application mein client-side routing implement karne mein help karti hai. Iska matlab hai ki aap bina page reload kiye different views/pages ke beech navigate kar sakte ho. Version 6 mein bahut saare improvements aur simplified APIs aaye hain jo development ko aur bhi easy banate hain.

Key Features:

  • Declarative Routing: Routes ko JSX components ki tarah define karo
  • Nested Routes: Routes ke andar routes banao
  • Dynamic Routing: URL parameters ke saath flexible routing
  • Code Splitting: Lazy loading se performance improve karo
  • Protected Routes: Authentication-based access control
üí° Pro Tip: React Router v6 ne v5 se kaafi changes kiye hain. Switch ab Routes ban gaya hai, aur component prop ab element prop ban gaya hai.

2Ô∏è‚É£ Nested Routes

Nested Routes ka matlab hai routes ke andar routes. Jaise agar aapke paas ek /dashboard route hai, aur uske andar /dashboard/profile aur /dashboard/settings chahiye, toh nested routes use karte hain. Isse aapka code organized aur maintainable rehta hai.
// Nested Routes Example <Routes> <Route path="/" element={<Home />} /> <Route path="/dashboard" element={<Dashboard />}> <Route path="profile" element={<Profile />} /> <Route path="settings" element={<Settings />} /> <Route path="analytics" element={<Analytics />} /> </Route> </Routes> // Dashboard.js - Parent component mein Outlet use karo import { Outlet } from 'react-router-dom'; function Dashboard() { return ( <div> <h1>Dashboard</h1> <Outlet /> {/* Nested routes yahan render honge */} </div> ); }

// Layout Component import { Outlet } from 'react-router-dom'; function Layout() { return ( <div> <Navbar /> <main> <Outlet /> {/* Child routes yahan render honge */} </main> <Footer /> </div> ); } // Routes with Layout <Routes> <Route element={<Layout />}> <Route path="/" element={<Home />} /> <Route path="/about" element={<About />} /> <Route path="/contact" element={<Contact />} /> </Route> </Routes>

4Ô∏è‚É£ Route Parameters (Dynamic Routes)

Route Parameters se aap dynamic URLs create kar sakte ho. Jaise /user/123 ya /product/laptop. Yeh parameters useParams hook se access kar sakte ho. Isse ek hi component multiple IDs ke liye reuse ho sakta hai.
// Route with parameter <Route path="/user/:userId" element={<UserProfile />} /> <Route path="/product/:productId" element={<ProductDetail />} /> // UserProfile.js - useParams hook se parameter access karo import { useParams } from 'react-router-dom'; function UserProfile() { const { userId } = useParams(); return ( <div> <h1>User Profile</h1> <p>User ID: {userId}</p> </div> ); }

Protected Routes wo routes hain jo sirf authenticated users access kar sakte hain. Agar user logged in nahi hai, toh use login page pe redirect kar dete hain. Isse aap apne sensitive pages ko secure kar sakte ho. Yeh authentication aur authorization ke liye bahut important hai.
// ProtectedRoute Component import { Navigate } from 'react-router-dom'; function ProtectedRoute({ children }) { const isAuthenticated = checkAuth(); // Your auth logic return isAuthenticated ? children : <Navigate to="/login" />; } // Usage in Routes <Routes> <Route path="/login" element={<Login />} /> <Route path="/dashboard" element={ <ProtectedRoute> <Dashboard /> </ProtectedRoute> } /> </Routes>

7Ô∏è‚É£ Lazy Loading Routes (Code Splitting)

Lazy Loading se aap apne components ko on-demand load kar sakte ho. Isse initial bundle size kam hota hai aur app faster load hoti hai. React ke lazy() aur Suspense ke saath use karte hain. Yeh performance optimization ke liye bahut important hai.
// Lazy loading with React.lazy() import { lazy, Suspense } from 'react'; const Home = lazy(() => import('./pages/Home')); const Dashboard = lazy(() => import('./pages/Dashboard')); const Profile = lazy(() => import('./pages/Profile')); function App() { return ( <Suspense fallback={<div>Loading...</div>}> <Routes> <Route path="/" element={<Home />} /> <Route path="/dashboard" element={<Dashboard />} /> <Route path="/profile" element={<Profile />} /> </Routes> </Suspense> ); }

8Ô∏è‚É£ Relative Link Paths

Relative Paths se aap current route ke relative links create kar sakte ho. Isse code maintainable rehta hai aur refactoring easy ho jata hai. Link aur useNavigate dono mein relative paths use kar sakte ho.
// Relative paths in nested routes import { Link, Outlet } from 'react-router-dom'; function Dashboard() { return ( <div> <nav> {/* Relative to current route */} <Link to="profile">Profile</Link> <Link to="settings">Settings</Link> {/* Go up one level */} <Link to="..">Back</Link> </nav> <Outlet /> </div> ); }

Q1: React Router v6 mein Switch component ko kya replace karta hai aur kyun?

Answer: React Router v6 mein Switch component ko Routes component ne replace kar diya hai. Yeh change isliye kiya gaya kyunki:

1. Better Naming: "Routes" naam zyada descriptive hai
2. Relative Routing: Routes component relative routing ko better support karta hai
3. Route Ranking: Automatic route ranking system hai jo best match find karta hai
4. Nested Routes: Nested routes ko handle karna zyada easy ho gaya hai

<Switch> (v5) ‚Üí <Routes> (v6)

Q2: useParams hook ka kya use hai? Example ke saath explain karo.

Answer: useParams hook URL parameters ko access karne ke liye use hota hai. Jab aap dynamic routes banate ho, tab yeh hook un parameters ko extract karta hai.

Example:
Route: /user/:userId/post/:postId
URL: /user/123/post/456

const { userId, postId } = useParams();
// userId = "123"
// postId = "456"
Use Cases: User profiles, product details, blog posts, etc.

Medium

Q4: Lazy loading routes se performance kaise improve hoti hai? Trade-offs kya hain?

Answer: Lazy loading se performance improvement:

Benefits:
1. Reduced Initial Bundle: Starting mein sirf required code load hota hai
2. Faster First Paint: Page jaldi render hota hai
3. Better Caching: Individual chunks cache ho sakte hain
4. On-Demand Loading: User jab route visit kare tabhi load ho

Trade-offs:
1. Loading Delay: First time route visit pe delay ho sakta hai
2. Network Requests: Extra network calls
3. Complexity: Error handling aur loading states manage karne padte hain
4. Suspense Required: Fallback UI provide karna zaroori hai

Best Practice: Heavy components (admin panels, dashboards, editors) ko lazy load karo, frequently used components ko nahi.
Medium

Q5: useSearchParams aur useParams mein kya difference hai? Kab kaunsa use karoge?

Answer:

useParams:
‚Ä¢ URL path segments se data extract karta hai
‚Ä¢ Route definition mein define hota hai: /user/:id
‚Ä¢ Read-only hai
‚Ä¢ Example: /user/123 ‚Üí { id: '123' }

useSearchParams:
‚Ä¢ Query strings se data extract karta hai
‚Ä¢ URL mein ? ke baad aata hai
‚Ä¢ Read aur Write dono kar sakte ho
‚Ä¢ Example: /products?sort=price&category=electronics

Use Cases:
‚Ä¢ useParams: Resource identification (user ID, product ID)
‚Ä¢ useSearchParams: Filtering, sorting, pagination, search queries

Example: /products/123?color=red&size=large
‚Ä¢ useParams() ‚Üí { productId: '123' }
‚Ä¢ useSearchParams() ‚Üí { color: 'red', size: 'large' }

Q7: React Router v6 mein route ranking kaise kaam karti hai? Ambiguous routes ko kaise handle karta hai?

Answer: React Router v6 automatic route ranking system use karta hai jo best match find karta hai:

Ranking Priority (High to Low):
1. Static segments: /about
2. Dynamic segments: /:id
3. Splat routes: /*
4. Index routes: index

Example:
<Routes>
  <Route path="/users/new" element={<NewUser />} />
  <Route path="/users/:id" element={<UserProfile />} />
  <Route path="/users/*" element={<UsersLayout />} />
</Routes>

‚Ä¢ /users/new ‚Üí NewUser (static match - highest priority)
‚Ä¢ /users/123 ‚Üí UserProfile (dynamic match)
‚Ä¢ /users/anything/else ‚Üí UsersLayout (splat match)

V5 vs V6:
‚Ä¢ V5 mein order matter karta tha (first match wins)
‚Ä¢ V6 mein specificity matter karti hai (best match wins)
‚Ä¢ V6 mein exact prop ki zaroorat nahi
