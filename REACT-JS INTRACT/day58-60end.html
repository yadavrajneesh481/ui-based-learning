name: Deploy to Vercel

on:
  push:
    branches: [ main ]

jobs:

    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Deploy to Vercel
        run: vercel --prod --token=${{ secrets.VERCEL_TOKEN }}

üéØ Interview Questions - Deployment & Production

Q1. What is the difference between development and production builds? Easy

Answer:

Development Build:

  • ‚Ä¢ Includes source maps for debugging
  • ‚Ä¢ Contains warning messages and error details
  • ‚Ä¢ Larger file size
  • ‚Ä¢ Hot Module Replacement (HMR) enabled
  • ‚Ä¢ Not optimized for performance

Production Build:

  • ‚Ä¢ Minified and compressed code
  • ‚Ä¢ Tree-shaking removes unused code
  • ‚Ä¢ Optimized for performance
  • ‚Ä¢ No development warnings
  • ‚Ä¢ Smaller bundle size

Development mein debugging easy hoti hai, production mein performance best hoti hai.

Q2. Explain environment variables and why they are important. Easy

Answer:

Environment variables are dynamic values that can change based on the environment (development, staging, production). They are important because:

  • ‚Ä¢ Security: Keep sensitive data (API keys, passwords) out of source code
  • ‚Ä¢ Flexibility: Different values for different environments
  • ‚Ä¢ Configuration: Easy to change settings without code changes
  • ‚Ä¢ Version Control: Secrets don't get committed to Git
// .env file
VITE_API_URL=https://api.production.com
VITE_API_KEY=secret_key_here

// Usage
const apiUrl = import.meta.env.VITE_API_URL;

Environment variables se aap secrets ko safe rakh sakte ho aur different environments ke liye alag settings use kar sakte ho.

Q3. What is Code Splitting and why is it beneficial? Medium

Answer:

Code Splitting is a technique where you split your application bundle into smaller chunks that can be loaded on demand.

Benefits:

  • ‚Ä¢ Faster Initial Load: Users download only what they need
  • ‚Ä¢ Better Performance: Reduced bundle size improves load time
  • ‚Ä¢ Lazy Loading: Load components only when needed
  • ‚Ä¢ Better Caching: Unchanged chunks remain cached
// React.lazy for code splitting
import React, { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./Dashboard'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Dashboard />
    </Suspense>
  );
}

Code splitting se aap apne app ko chhote pieces mein tod sakte ho, jisse initial load time kam ho jata hai.

Q4. Explain CI/CD and its importance in modern development. Medium

Answer:

CI (Continuous Integration): Automatically building and testing code whenever changes are pushed to the repository.

CD (Continuous Deployment): Automatically deploying tested code to production.

Importance:

  • ‚Ä¢ Automation: Reduces manual deployment errors
  • ‚Ä¢ Faster Releases: Deploy multiple times per day
  • ‚Ä¢ Quality Assurance: Automated testing catches bugs early
  • ‚Ä¢ Rollback: Easy to revert to previous versions
  • ‚Ä¢ Consistency: Same process every time
# GitHub Actions workflow
name: CI/CD Pipeline
on: [push]
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
      - name: Install dependencies
      - name: Run tests
      - name: Build
      - name: Deploy

CI/CD se development process automatic ho jata hai - code push karo aur automatically test, build, deploy ho jata hai.

Q6. How do you optimize a React app for production? Hard

Answer:

Performance Optimization Techniques:

  • ‚Ä¢ Code Splitting: Use React.lazy() and dynamic imports
  • ‚Ä¢ Memoization: Use React.memo, useMemo, useCallback
  • ‚Ä¢ Image Optimization: Use WebP format, lazy loading, responsive images
  • ‚Ä¢ Bundle Analysis: Use webpack-bundle-analyzer to identify large dependencies
  • ‚Ä¢ Tree Shaking: Remove unused code from final bundle
  • ‚Ä¢ Compression: Enable Gzip/Brotli compression
  • ‚Ä¢ CDN: Serve static assets from CDN
  • ‚Ä¢ Caching: Implement proper cache headers
  • ‚Ä¢ Prefetching: Preload critical resources
// Memoization example
const MemoizedComponent = React.memo(({ data }) => {
  return <div>{data}</div>;
});

// useMemo for expensive calculations
const expensiveValue = useMemo(() => {
  return computeExpensiveValue(data);

// Image lazy loading

Production optimization mein code splitting, memoization, image optimization, aur caching sabse important hain.

Q7. Explain the concept of Tree Shaking and how it works. Hard

Answer:

Tree Shaking is a dead code elimination technique that removes unused code from the final bundle during the build process.

How it works:

  • ‚Ä¢ Static Analysis: Build tool analyzes import/export statements
  • ‚Ä¢ ES6 Modules: Requires ES6 module syntax (import/export)
  • ‚Ä¢ Dead Code Detection: Identifies code that's never used
  • ‚Ä¢ Elimination: Removes unused code from bundle
// utils.js
export function usedFunction() { return 'used'; }
export function unusedFunction() { return 'unused'; }

// app.js
import { usedFunction } from './utils';
// unusedFunction is NOT imported

// After tree shaking, unusedFunction is removed from bundle

Requirements for Tree Shaking:

  • ‚Ä¢ Use ES6 module syntax (not CommonJS)
  • ‚Ä¢ Avoid side effects in modules
  • ‚Ä¢ Use production mode build
  • ‚Ä¢ Configure sideEffects in package.json

Tree shaking se unused code automatically remove ho jata hai, jisse bundle size kam ho jata hai aur performance improve hoti hai.
