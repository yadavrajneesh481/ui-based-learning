1Ô∏è‚É£ Global CSS - Traditional Approach

Ye sabse purana aur simple tarika hai! Isme hum normal CSS file banate hain aur use import kar lete hain. Jaise traditional websites mein karte the, waise hi. Lekin iska ek problem hai - saare styles global ho jate hain, matlab kisi bhi component mein apply ho sakte hain.

Global CSS: A traditional styling approach where CSS rules are defined in separate .css files and imported into components, making styles available globally across the application. 
/* styles.css */
.button {
    background-color: #61dafb;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    cursor: pointer;
}

.button:hover {
    background-color: #4fa8c5;
}

/* App.js */
import './styles.css';

function App() {
    return <button className="button">Click Me</button>;
}

2Ô∏è‚É£ CSS Modules - Scoped Styling

CSS Modules ek game-changer hai! Isme har component ki apni CSS file hoti hai aur wo automatically scoped ho jati hai. Matlab ek component ki styling dusre component ko affect nahi karegi. Ye Create React App mein by default supported hai!

CSS Modules: A CSS file in which all class names and animation names are scoped locally by default, preventing naming conflicts and ensuring component-level encapsulation. 
/* Button.module.css */

.button:hover {

    transform: scale(1.05);
}

/* Button.js */
import styles from './Button.module.css';

function Button() {
    return <button className={styles.button}>Click Me</button>;
}

3Ô∏è‚É£ Styled Components - CSS-in-JS Magic

Ye sabse popular CSS-in-JS library hai! Isme tum JavaScript ke andar hi CSS likh sakte ho. Bahut powerful hai kyunki tum props ke basis pe dynamic styling kar sakte ho. Tagged template literals ka use karte hain isme.

Styled Components: A CSS-in-JS library that allows you to write actual CSS code to style components using tagged template literals, enabling dynamic styling based on props and automatic critical CSS injection. 
import styled from 'styled-components';

// Styled component create karo
const Button = styled.button`
    background-color: ${props => props.primary ? '#61dafb' : '#gray'};
    color: white;
    padding: 12px 24px;
    border: none;
    border-radius: 8px;
    font-size: 16px;
    cursor: pointer;
    transition: all 0.3s ease;

    &:hover {
        transform: translateY(-2px);
        box-shadow: 0 5px 15px rgba(0,0,0,0.2);
    }
`;

// Use karo
function App() {
    return (
        <>
            <Button primary>Primary Button</Button>
            <Button>Secondary Button</Button>
        </>
    );
}

4Ô∏è‚É£ Emotion - Performant CSS-in-JS

Emotion bhi ek powerful CSS-in-JS library hai! Ye styled-components jaisi hai lekin thodi zyada flexible aur performant. Isme do tarike hain styling karne ke - styled API aur css prop. Bahut fast hai aur bundle size bhi chota hai!

Emotion: A performant and flexible CSS-in-JS library that provides both styled component API and css prop approach, with excellent performance characteristics and smaller bundle size compared to alternatives. 
/** @jsxImportSource @emotion/react */

import styled from '@emotion/styled';

// Method 1: css prop

function App() {

// Method 2: styled API

Tailwind CSS: A utility-first CSS framework that provides low-level utility classes to build custom designs without writing custom CSS, promoting rapid development and consistent design systems.

Dark Mode: A color scheme that uses light-colored text and UI elements on dark backgrounds, reducing eye strain in low-light conditions and potentially saving battery on OLED screens. 
// CSS Variables Approach

Q1: What is the difference between CSS Modules and regular CSS? Easy

Answer:

Regular CSS: Global scope mein hota hai, saare classes globally available hote hain, naming conflicts ho sakte hain.

CSS Modules: Local scope mein hota hai, har component ki apni scoped classes hoti hain, automatically unique class names generate hote hain.

// Regular CSS
import './styles.css';
<div className="button">Click</div>

// CSS Modules
import styles from './Button.module.css';
<div className={styles.button}>Click</div>

Key Point: CSS Modules prevent naming collisions and provide component-level encapsulation.

CSS-in-JS: JavaScript ke andar CSS likhne ka approach hai. Libraries like styled-components aur Emotion iska use karti hain.

Advantages:

Q4: What are the performance implications of different styling approaches? Hard

Answer:

Performance Comparison:

  • Global CSS: Fastest - static files, browser caching, no runtime overhead
  • CSS Modules: Fast - compile-time processing, no runtime cost
  • Styled Components: Small runtime overhead - styles generate hote hain runtime pe, but critical CSS injection se optimize hota hai
  • Tailwind CSS: Fast after purging - unused classes remove ho jate hain build time pe

Optimization Tips:

  • ‚úÖ Code splitting use karo
  • ‚úÖ Critical CSS inline karo
  • ‚úÖ Unused styles purge karo
  • ‚úÖ CSS minification enable karo
  • ‚úÖ Server-side rendering consider karo

Q7: What factors should you consider when choosing a styling solution for a React project? Hard

Answer:

Key Factors to Consider:

  • 1. Project Size: Small projects ke liye simple CSS, large projects ke liye CSS-in-JS ya Tailwind
  • 2. Team Experience: Team ko kya pata hai? Traditional CSS ya modern approaches?
  • 3. Performance Requirements: Critical performance needs? Static CSS best hai
  • 4. Dynamic Styling Needs: Bahut dynamic styling chahiye? CSS-in-JS use karo
  • 5. Bundle Size: Bundle size important hai? Emotion ya Tailwind consider karo
  • 6. Development Speed: Fast development? UI libraries ya Tailwind use karo
  • 7. Maintenance: Long-term maintenance? CSS Modules ya styled-components
  • 8. SSR Requirements: Server-side rendering? CSS-in-JS libraries SSR support karte hain

Decision Matrix:

  • üéØ Simple Project: Global CSS or CSS Modules
  • üéØ Component Library: Styled-components or Emotion
  • üéØ Rapid Development: Tailwind CSS or UI Library
  • üéØ Enterprise App: CSS Modules + Design System
